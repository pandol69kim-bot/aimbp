### W27 — 구독 플랜 & 결제 연동 (27주차)

#### Backend
- [ ] SubscriptionPlan 모델 설계
  - [ ] id, name (Free/Creator/Studio/Label), price, monthly_quota, features
  - [ ] Free: 5곡/월, Creator: 100곡/월, Studio: 무제한, Label: 다중계정+팀협업
- [ ] UserSubscription 모델
  - [ ] id, user_id, plan_id, status, started_at, expires_at, auto_renew
- [ ] 결제 게이트웨이 연동
  - [ ] Stripe API 클라이언트 구현
  - [ ] 토스페이먼츠 API 클라이언트 구현 (국내)
- [ ] 구독 결제 API `POST /subscriptions/checkout`
  - [ ] 플랜 선택 → 결제창 세션 생성
  - [ ] Webhook 수신 → 구독 활성화
- [ ] 구독 갱신 API `POST /subscriptions/renew`
- [ ] 구독 취소 API `POST /subscriptions/cancel`
- [ ] 구독 조회 API `GET /subscriptions/me`
- [ ] 월별 생성 쿼터 카운터 모듈
  - [ ] 생성 요청 시 쿼터 차감
  - [ ] 쿼터 초과 시 업그레이드 유도 응답

#### Frontend
- [ ] 구독 플랜 비교 페이지 (4개 플랜 카드 UI)
- [ ] 결제 체크아웃 페이지 (Stripe Elements / 토스페이먼츠 위젯)
- [ ] 구독 현황 페이지 (플랜명·만료일·쿼터 잔여량)
- [ ] 쿼터 초과 시 업그레이드 유도 모달
- [ ] 구독 취소 확인 UI (해지 사유 수집)
- [ ] 결제 영수증 다운로드

#### 리뷰 체크
- [ ] 결제 금액 서버 측 검증 확인 (클라이언트 금액 신뢰 금지)
- [ ] Webhook 서명 검증 처리 확인 (Stripe signature)
- [ ] 결제 실패 시 구독 활성화 차단 확인
- [ ] 이중 결제 방지 로직 확인 (멱등 키)
- [ ] PCI DSS 준수 여부 확인 (카드 정보 직접 저장 금지)
- [ ] 쿼터 동시 차감 Race Condition 방지 확인
- [ ] 구독 만료 전 갱신 알림 이메일 발송 확인 (D-7, D-1)

### W28 — 출금 처리 & 정산 관리 (28주차)

#### Backend
- [ ] 출금 계좌 모델 (id, user_id, bank_code, account_no_encrypted, holder_name, verified)
- [ ] 계좌 실명 인증 API `POST /payouts/verify-account` (금융결제원 또는 토스 API)
- [ ] 출금 신청 API `POST /payouts/request`
  - [ ] 최소 출금 금액 정책 적용 (예: 1만 원 이상)
  - [ ] 출금 가능 잔액 계산 로직
- [ ] 출금 내역 조회 API `GET /payouts`
- [ ] 출금 처리 배치 (주 1회 또는 월 2회 자동 이체)
- [ ] Payout 모델 (id, user_id, amount, status, requested_at, paid_at, tx_ref)
- [ ] 플랫폼 수수료 계산 모듈
  - [ ] 유통 수수료 (%), 플랫폼 수수료 차감 후 정산
- [ ] 연간 정산 내역 PDF 생성 (세금계산서 기초 자료)

#### Frontend
- [ ] 출금 계좌 등록·수정 UI
- [ ] 계좌 실명 인증 완료 배지
- [ ] 출금 신청 폼 (금액 입력 + 계좌 선택)
- [ ] 출금 이력 테이블 (상태별 필터: 신청/처리중/완료/실패)
- [ ] 수수료 차감 내역 상세 보기
- [ ] 연간 정산서 다운로드 버튼

#### 리뷰 체크
- [ ] 계좌번호 AES-256 암호화 저장 확인
- [ ] 출금 신청 시 잔액 음수 방지 확인
- [ ] 출금 이중 신청 방지 확인 (pending 상태 중복 차단)
- [ ] 배치 실패 시 수동 재처리 관리자 기능 확인
- [ ] 개인정보처리방침 내 계좌 정보 보존 기간 준수 확인

### W29 — 관리자 패널 (29주차)

#### Backend — 사용자 관리
- [ ] 관리자 권한 미들웨어 (role: admin)
- [ ] 사용자 목록 조회 API `GET /admin/users`
  - [ ] 필터: 가입일, 플랜, 상태 (활성/정지/탈퇴)
  - [ ] 검색: 이메일, 닉네임
- [ ] 사용자 상세 조회 API `GET /admin/users/{id}`
- [ ] 사용자 플랜 변경 API `PATCH /admin/users/{id}/plan`
- [ ] 사용자 정지/복구 API `PATCH /admin/users/{id}/status`

#### Backend — 정산 관리
- [ ] 전체 출금 신청 목록 API `GET /admin/payouts`
- [ ] 출금 승인/거절 API `PATCH /admin/payouts/{id}/approve`
- [ ] 플랫폼 수수료 설정 API `PATCH /admin/settings/fee`
- [ ] 월별 정산 통계 API `GET /admin/revenue/stats`

#### Backend — 콘텐츠 검수
- [ ] 검수 대기 콘텐츠 목록 API `GET /admin/content/review`
- [ ] 콘텐츠 승인/거절 API `PATCH /admin/content/{id}/status`
- [ ] 검수 사유 메모 저장

#### Frontend
- [ ] 관리자 전용 레이아웃 (어드민 사이드바)
- [ ] 사용자 관리 테이블 (정렬·필터·검색)
- [ ] 사용자 상세 모달 (구독·생성 이력·출금 이력)
- [ ] 출금 승인 처리 UI (일괄 승인 지원)
- [ ] 수익 통계 대시보드 (MRR, 신규 가입, 플랜별 분포)
- [ ] 콘텐츠 검수 큐 UI (승인/거절 버튼 + 사유 입력)

#### 리뷰 체크
- [ ] 관리자 API 일반 사용자 접근 차단 확인
- [ ] 관리자 작업 감사 로그 저장 확인 (누가·언제·무엇을)
- [ ] 사용자 개인정보 마스킹 처리 확인 (이메일·계좌 일부 숨김)
- [ ] 일괄 처리 작업 실패 부분 롤백 확인
- [ ] 관리자 계정 2FA 적용 확인

### W30 — AI 생성 검증 & 전체 통합 테스트 & 론칭 준비 (30주차)

#### AI 생성 검증

##### Backend
- [ ] 저작권 검사 모듈
  - [ ] 외부 저작권 DB 조회 (ACRCloud 또는 AudD API)
  - [ ] 유사도 임계값 설정 (85% 이상 시 경고)
  - [ ] 검사 결과 저장 (CopyrightCheck 모델)
- [ ] AI 생성 여부 검증 모듈
  - [ ] 생성 로그 메타데이터 기반 AI 생성 여부 판정
  - [ ] 수동 업로드 음원 AI 생성 여부 분석 (선택)
- [ ] 정책 위반 감지 모듈
  - [ ] 혐오 발언 / 성인 콘텐츠 키워드 필터 (가사 분석)
  - [ ] 콘텐츠 정책 위반 시 자동 검수 대기 전환
- [ ] 검증 API `POST /content/{id}/validate`
- [ ] 검증 결과 조회 API `GET /content/{id}/validation`

##### Frontend
- [ ] 업로드/생성 시 자동 검증 진행 표시
- [ ] 검증 결과 배지 (통과 / 경고 / 차단)
- [ ] 저작권 유사 곡 목록 표시 (경고 시)
- [ ] 이의 신청 UI (차단된 콘텐츠 검토 요청)

##### 리뷰 체크
- [ ] 저작권 검사 오탐률 기준 확인 (5% 이하)
- [ ] 검사 API 응답 시간 확인 (30초 이내)
- [ ] 검증 실패 시 사용자 명확한 안내 확인
- [ ] 이의 신청 처리 관리자 알림 확인

#### Phase 5 통합 테스트

- [ ] 신규 가입 → 무료 플랜 → 쿼터 소진 → 유료 업그레이드 → 음원 생성 → 유통 → 수익 수집 → 출금 신청 전체 E2E
- [ ] 관리자: 사용자 정지 → 해당 사용자 접근 차단 시나리오
- [ ] 저작권 위반 음원 업로드 → 자동 차단 → 이의 신청 → 관리자 검토 시나리오
- [ ] 구독 만료 → 쿼터 초기화 → 갱신 시나리오
- [ ] 동시 사용자 100명 기준 부하 테스트

---

#### 론칭 준비 체크리스트

##### 보안 최종 점검
- [ ] 전체 API 인증·인가 누락 엔드포인트 없음 확인
- [ ] 하드코딩된 시크릿 키 없음 확인 (git-secrets 스캔)
- [ ] HTTPS 강제 적용 (HTTP → HTTPS 리디렉션)
- [ ] Security Headers 설정 확인 (HSTS, CSP, X-Frame-Options)
- [ ] Rate Limiting 전 엔드포인트 적용 확인
- [ ] SQL Injection / XSS 취약점 스캔 통과 확인
- [ ] 개인정보처리방침 및 이용약관 페이지 게시 확인

##### 인프라 최종 점검
- [ ] AWS 프로덕션 환경 구성 (ECS 또는 EKS)
- [ ] RDS PostgreSQL 자동 백업 활성화 (일 1회)
- [ ] S3 버킷 버전 관리 활성화
- [ ] CloudWatch 알람 설정 (CPU 80%, 메모리 80%, 에러율 5%)
- [ ] 도메인 연결 및 SSL 인증서 발급 확인
- [ ] CDN 설정 (CloudFront)

##### 품질 최종 점검
- [ ] 전체 테스트 커버리지 80% 이상 확인
- [ ] 브라우저 호환성 확인 (Chrome / Firefox / Safari)
- [ ] 모바일 반응형 확인 (320px / 375px / 768px)
- [ ] Lighthouse 점수 확인 (Performance 80+, Accessibility 90+)
- [ ] 콘솔 에러 없음 확인

##### 운영 준비
- [ ] 오류 모니터링 설정 (Sentry 또는 동급)
- [ ] 로그 수집 설정 (CloudWatch Logs)
- [ ] 서비스 장애 대응 플레이북 작성
- [ ] 고객 지원 채널 개설 (이메일 / 채팅)
- [ ] 베타 테스터 10명 이상 확보 및 피드백 수집
