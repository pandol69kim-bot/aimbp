## 작업 내용

앨범 상세 페이지의 "커버 생성" 버튼을 "생성된 커버 가져오기"로 교체했습니다.
커버 생성 페이지에서 만든 커버를 앨범에 바로 적용할 수 있습니다.

## 변경된 파일

### `frontend/src/app/(dashboard)/albums/[id]/page.tsx`
- "커버 생성" 버튼 및 생성 다이얼로그 제거
- `generateCoverMutation`, `coverGenre/Mood/Keywords` 상태 제거
- 앨범별 커버 생성 이력 섹션 제거
- **"생성된 커버 가져오기"** 버튼 + 픽커 다이얼로그 신규 추가
- 선택한 이미지 URL → `updateAlbum({ cover_url })` 으로 앨범 커버 즉시 적용

### `frontend/src/types/index.ts`
- `CoverImage` 인터페이스를 실제 백엔드 응답에 맞게 수정
  - `prompt_genre`, `prompt_mood`, `prompt_keywords`, `image_url`, `size`, `album_id` 추가
  - 기존 `genre`, `mood`, `keywords` 하위호환 optional로 유지

## 주요 동작

1. 앨범 상세 → "생성된 커버 가져오기" 클릭
2. 다이얼로그: 완료(completed)된 커버를 프롬프트별로 그룹화해 표시
3. 이미지 클릭 → 체크 아이콘 + 보라색 링으로 선택 표시
4. "앨범에 적용" 클릭 → `album.cover_url` 업데이트 → 헤더 커버 즉시 반영
5. 커버 없을 시 "커버 생성 페이지에서 먼저 만들어주세요" 안내

## 배포

```bash
docker-compose build --no-cache frontend
docker-compose up -d frontend
```
