# 앨범 커버 생성(Cover Generation) 프로세스 정의

## 개요

앨범 커버 생성은 AI 기술을 활용하여 사용자가 지정한 장르, 분위기, 키워드에 맞춰 3가지 비율의 앨범 커버 이미지를 자동으로 생성하는 프로세스입니다.

---

## 1. 생성 가능한 비율(Aspect Ratios)

### 3가지 표준 비율

| 비율 | 크기 | 용도 | 해상도 |
|------|------|------|--------|
| **1:1** | 정사각형 | 앨범 커버, SNS 프로필 | 1024x1024 (DALL-E 3) |
| **16:9** | 가로형 | 유튜브 썸네일, 배너 | 1792x1024 또는 1536x1024 |
| **9:16** | 세로형 | 스토리, 모바일 배경화면 | 1024x1792 또는 1024x1536 |

### 모델별 해상도 맵

**DALL-E 3**
```
1:1   → 1024x1024
16:9  → 1792x1024
9:16  → 1024x1792
```

**GPT Image 1**
```
1:1   → 1024x1024
16:9  → 1536x1024
9:16  → 1024x1536
```

**Stable Diffusion (로컬)**
```
1:1   → 512x512
16:9  → 1024x576
9:16  → 576x1024
```

## 2. AI 모델 선택

### 가용 모델

| 모델 | 특징 | 비용 | 품질 | 속도 |
|------|------|------|------|------|
| **GPT Image 1** | 최신 모델, 고품질 | 높음 | ⭐⭐⭐⭐⭐ | 보통 |
| **DALL-E 3** | 안정적, 널리 사용 | 중간 | ⭐⭐⭐⭐ | 보통 |
| **Stable Diffusion** | 로컬 실행 가능, 무료 | 낮음 | ⭐⭐⭐ | 빠름 |

### 모델 선택 기준

| 우선순위 | 모델 | 이유 |
|---------|------|------|
| 1순위 | GPT Image 1 | OpenAI API 활성화 시 자동 선택 |
| 2순위 | DALL-E 3 | OpenAI API 사용 불가 시 대체 |
| 3순위 | Stable Diffusion | 로컬 생성 (비용 없음) |
| 최후의 수단 | Mock 이미지 | 모든 생성 실패 시 |

## 4. 생성 프로세스 흐름

### Step 1: 사용자 입력

```
사용자가 커버 생성 페이지 접속
    ↓
장르, 분위기, 키워드 선택 및 입력
    ↓
AI 모델 선택 (선택사항, 기본값: GPT Image 1)
    ↓
"커버 생성하기" 버튼 클릭
```

### Step 2: 요청 전송

```
클라이언트 (Frontend)
├─ 입력값 검증 (모든 필드 필수)
├─ API 요청 생성
│  POST /api/v1/cover/generate
│  {
│    "prompt_genre": "pop",
│    "prompt_mood": "energetic",
│    "prompt_keywords": "야경, 도시, 별빛",
│    "ai_model": "gpt-image-1"
│  }
└─ 로딩 상태 표시
```

### Step 3: 백엔드 처리

```
서버 (Backend)
├─ 요청 수신 및 검증
├─ 3개의 Cover 레코드 생성 (1:1, 16:9, 9:16)
│  ├─ size: "1:1"
│  ├─ size: "16:9"
│  └─ size: "9:16"
├─ 각 Cover를 DB에 저장 (status: pending)
├─ 백그라운드 작업 3개 예약
└─ 즉시 응답 반환 (status: pending)
```

### Step 4: 비동기 이미지 생성

```
백그라운드 작업 (각 사이즈별 병렬 처리)
├─ _process_cover(cover_1x1_id)
│  ├─ generate_cover_image(...)
│  │  ├─ GPT Image 1 시도
│  │  ├─ DALL-E 3 시도
│  │  └─ Stable Diffusion 시도
│  ├─ 이미지를 파일로 저장
│  └─ DB 업데이트 (status: completed, image_url)
├─ _process_cover(cover_16x9_id)
│  └─ 동일 처리
└─ _process_cover(cover_9x16_id)
   └─ 동일 처리
```

### Step 5: 프론트엔드 실시간 업데이트

```
프론트엔드 (폴링 또는 웹소켓)
├─ 5초마다 GET /api/v1/cover 호출
├─ 새 커버 데이터 감지
├─ UI 업데이트
│  ├─ 상태: pending → completed
│  ├─ 이미지 로드 및 표시
│  └─ 다운로드 버튼 활성화
└─ 생성 완료 알림
```

## 5. AI 모델 선택 로직 (Fallback Chain)

```python
async def generate_cover_image(
    prompt_genre,
    prompt_mood, 
    prompt_keywords,
    size="1:1",
    ai_model="gpt-image-1"
):
    """
    우선순위:
    1. 사용자가 선택한 AI 모델
    2. 사용 가능한 대체 모델
    3. 로컬 생성 (Stable Diffusion)
    4. Mock 이미지
    """
    
    # Step 1: GPT Image 1 시도
    if ai_model == "gpt-image-1" and settings.has_openai:
        result = await _generate_gpt_image_1(...)
        if result:
            return result
    
    # Step 2: DALL-E 3 시도
    if ai_model in ("dalle-3", "gpt-image") and settings.has_openai:
        result = await _generate_openai_image(...)
        if result:
            return result
    
    # Step 3: Stable Diffusion 시도
    result = await _generate_stable_diffusion_image(...)
    if result:
        return result
    
    # Step 4: Mock 이미지 반환
    logger.info(f"Using mock cover image for size {size}")
    return MOCK_COVER_URLS.get(size)
```

async def _generate_gpt_image_1(
    prompt_genre, prompt_mood, prompt_keywords, size

### 6.3 Stable Diffusion (로컬)

```python
async def _generate_stable_diffusion_image(
    prompt_genre, prompt_mood, prompt_keywords, size
):
    """PIL로 그래디언트 이미지 생성"""
    try:
        from PIL import Image, ImageDraw
        
        # 사이즈 맵핑
        size_map = {"1:1": (512, 512), "16:9": (1024, 576), "9:16": (576, 1024)}
        width, height = size_map.get(size, (512, 512))
        
        # 분위기별 색상
        mood_colors = {
            "peaceful": ((100, 150, 200), (150, 200, 240)),
            "dark": ((20, 20, 40), (50, 50, 80)),
            "energetic": ((255, 100, 50), (255, 200, 100)),
            "romantic": ((200, 100, 150), (255, 150, 200)),
            "melancholic": ((80, 80, 120), (120, 120, 160)),
        }
        
        start_color, end_color = mood_colors.get(
            prompt_mood, ((100, 100, 150), (150, 150, 200))
        )
        
        # 그래디언트 배경 생성
        image = Image.new("RGB", (width, height), start_color)
        draw = ImageDraw.Draw(image)
        
        for y in range(height):
            ratio = y / height
            r = int(start_color[0] * (1 - ratio) + end_color[0] * ratio)
            g = int(start_color[1] * (1 - ratio) + end_color[1] * ratio)
            b = int(start_color[2] * (1 - ratio) + end_color[2] * ratio)
            draw.line([(0, y), (width, y)], fill=(r, g, b))
        
        # Base64로 변환
        buffer = io.BytesIO()
        image.save(buffer, format="PNG")
        img_bytes = buffer.getvalue()
        b64_string = base64.b64encode(img_bytes).decode()
        
        return f"data:image/png;base64,{b64_string}"
    
    except Exception as e:
        logger.warning(f"Local generation failed: {e}")
        return None
```

## 7. 이미지 저장 프로세스

### 7.1 Base64 데이터 저장

```python
async def _save_cover_to_file(cover_id: str, size: str, image_url: str) -> str:
    """
    Base64 데이터URL을 파일로 저장하고 접근 가능한 URL 반환
    """
    # URL 형식이면 그대로 반환
    if not image_url.startswith("data:image/"):
        return image_url
    
    try:
        # Base64 디코딩
        _, b64_data = image_url.split(",", 1)
        img_bytes = base64.b64decode(b64_data)
        
        # 파일 저장
        covers_dir = "/app/uploads/covers"
        os.makedirs(covers_dir, exist_ok=True)
        size_key = size.replace(":", "x")
        filename = f"cover_{cover_id}_{size_key}.png"
        
        with open(f"{covers_dir}/{filename}", "wb") as f:
            f.write(img_bytes)
        
        # 접근 가능한 URL 생성
        url = f"{settings.FILE_BASE_URL}/api/v1/files/local/covers/{filename}"
        return url
    
    except Exception as e:
        logger.warning(f"Failed to save, keeping base64: {e}")
        return image_url
```

## 20. 자주 묻는 질문 (FAQ)

**Q: 커버 생성 시간은?**
A: 보통 1-2분, 로컬 생성은 10-30초

**Q: 생성된 이미지를 상용으로 사용 가능한가?**
A: 사용 가능하지만 라이센스 확인 권장

**Q: 같은 프롬프트로 다시 생성할 수 있나?**
A: 가능, 다시 요청하면 새 이미지 생성

**Q: 이미지를 직접 수정할 수 있나?**
A: 현재는 불가, 재생성 후 새 이미지 선택

**Q: 생성된 이미지는 어디에 저장되나?**
A: `/app/uploads/covers/` 또는 Base64 형식
