## 작업 내용

마스터링 시작 시 발생하는 `AxiosError: 400 Bad Request` 오류를 수정했습니다.

## 원인

두 가지 불일치 발견:

### 1. 플랫폼 값 불일치 (400 직접 원인)

프론트엔드가 `youtube_music`을 전송하지만 백엔드 허용 목록에 없었음.

| | 프론트 전송값 | 백엔드 허용값 |
|---|---|---|
| ✅ | `spotify` | `spotify` |
| ✅ | `apple_music` | `apple_music` |
| ❌ | `youtube_music` | `youtube` ← 불일치 |

### 2. 응답 필드명 불일치 (다운로드 버튼 미표시 원인)

| 백엔드 응답 | 프론트 기대값 |
|---|---|
| `input_url` | `original_url` |
| `output_url` | `mastered_url` |

## 수정된 파일

### `backend/app/api/mastering.py`
```python
# 변경 전
VALID_PLATFORMS = {"spotify", "apple_music", "youtube", "soundcloud", "tidal"}

# 변경 후
VALID_PLATFORMS = {"spotify", "apple_music", "youtube_music"}
```

### `backend/app/schemas/mastering.py`
- `MasteringResponse` 필드명 변경: `input_url` → `original_url`, `output_url` → `mastered_url`
- `model_validator`로 ORM 객체 → 응답 필드명 자동 매핑 처리

## 검증

```
POST /api/v1/mastering/process
{ "track_id": "...", "platform": "youtube_music" }
→ HTTP 201 ✅
→ 응답: { "original_url": "", "mastered_url": null, ... }
```

## 배포

```bash
docker-compose build --no-cache backend
docker-compose up -d backend
```
