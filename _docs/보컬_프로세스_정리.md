# 보컬(Vocal) 프로세스 정리

## 개요

보컬은 생성된 트랙에 AI 성우 보컬을 추가하여 완성도 있는 음악을 만드는 프로세스입니다.

## 2. 데이터 모델

### VocalLibrary (보컬 라이브러리)
```python
# 위치: backend/app/models/vocal.py

class VocalLibrary(Base):
    id: UUID (Primary Key)
    name: str                    # 보컬명 (Aria, Noah, Luna, Jay)
    gender: str                  # 남성(male) / 여성(female) / 중성(neutral)
    genre: str                   # 장르 (K-Pop, Rock, EDM, Rap, ...)
    sample_url: str              # 샘플 음성 URL
    is_active: bool              # 활성화 여부
```

### Vocal (생성된 보컬)
```python
# 위치: backend/app/models/vocal.py

class Vocal(Base):
    id: UUID (Primary Key)
    track_id: UUID (FK → tracks)      # 대상 트랙
    user_id: UUID (FK → users)        # 소유 사용자
    library_id: UUID (FK → vocal_library)  # 선택된 보컬 라이브러리
    language: str                     # 언어 (korean, english, japanese)
    file_url: str | None              # 생성된 보컬 파일 URL
    file_key: str | None              # S3 파일 키
    status: str                       # pending / completed / failed
    created_at: datetime              # 생성 시간
```

## 4. 백엔드 처리 흐름

### API 엔드포인트 (api/vocal.py)

#### Step 1: 요청 검증
```python
@router.post("/generate")
async def generate_vocal(payload: VocalCreate, current_user, db):
    # 1. 트랙 소유권 검증
    track = db.query(Track).where(
        Track.id == payload.track_id,
        Track.user_id == current_user.id
    )
    if not track:
        raise 404 "Track not found"
    
    # 2. 보컬 라이브러리 검증
    library = db.query(VocalLibrary).where(
        VocalLibrary.id == payload.library_id,
        VocalLibrary.is_active == True
    )
    if not library:
        # Mock ID도 허용
        if str(payload.library_id) not in mock_ids:
            raise 404 "Vocal library entry not found"
    
    # 3. Vocal 레코드 생성
    vocal = Vocal(
        track_id=payload.track_id,
        user_id=current_user.id,
        library_id=payload.library_id,
        language=payload.language,
        status="pending"
    )
    db.add(vocal)
    db.flush()
```

#### Step 2: 백그라운드 작업 시작
```python
    # 4. 백그라운드 작업 예약
    start_vocal_processing(
        vocal.id,
        background_tasks,
        db_session_factory
    )
    
    # 5. 즉시 응답
    return { "status": "pending", ... }
```

## 6. 상태 관리

### Track 상태 (보컬 적용 가능한 조건)
```
pending       ❌ 보컬 불가
processing    ❌ 보컬 불가
completed     ✅ 보컬 가능
failed        ❌ 보컬 불가
```

### Vocal 상태
```
pending    → 처리 대기 중
completed  → 완료, 보컬 파일 생성됨
failed     → 처리 실패 (미구현)
```

## 15. API 오류 처리

### 422 Unprocessable Entity (필드 검증 실패)
```
원인:
- track_id 누락 또는 형식 오류
- library_id 누락 또는 형식 오류
- language 형식 오류

해결:
- 모든 필드이 필수입니다
- UUID 형식을 확인하세요
- language는 'korean', 'english', 'japanese' 중 하나여야 합니다
```

### 404 Not Found (리소스 없음)
```
원인:
- 트랙이 없거나 소유권이 없음
- 보컬 라이브러리 ID가 잘못됨

해결:
- 완료된 트랙만 선택하세요
- 보컬 라이브러리에서 선택한 보컬의 ID를 확인하세요
```

## 17. 디버깅 팁

### 보컬 생성이 안 될 때
```
1. 브라우저 개발자 도구 → Network 탭 확인
2. POST /vocal/generate 요청 응답 확인
3. 422 오류면 요청 본문 필드 확인
4. 400 오류면 트랙/보컬 ID 확인
```

### 보컬 목록이 안 보일 때
```
1. GET /vocal 요청이 성공했는지 확인
2. 응답 데이터 구조 확인 (success, data, error)
3. 백엔드 로그 확인: docker-compose logs backend
```

### 상태가 업데이트 안 될 때
```
1. 폴링 간격 확인 (5초)
2. 백엔드 백그라운드 작업 실행 확인
3. DB에 데이터가 저장되었는지 확인
```
