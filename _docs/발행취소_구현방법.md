## 📋 개요

현재 상태:
- ✅ 앨범 **발행** 가능 (draft → published)
- ❌ 앨범 **발행 취소** 불가능

요구사항:
- 사용자가 발행된 앨범을 다시 draft 상태로 돌릴 수 있어야 함
- 발행 취소 후 앨범 수정 가능해야 함

### Phase 1: 백엔드 API 추가

#### 1.1 API 엔드포인트

**경로**: `backend/app/api/albums.py`

```python
@router.post("/{album_id}/unpublish")
async def unpublish_album(
    album_id: UUID,
    current_user: User = Depends(get_current_user),
    db: AsyncSession = Depends(get_db),
):
    """Unpublish album and revert to draft state."""
    album = await db.get(Album, album_id)
    
    if not album:
        raise HTTPException(
            status_code=status.HTTP_404_NOT_FOUND,
            detail="Album not found"
        )
    
    if album.user_id != current_user.id:
        raise HTTPException(
            status_code=status.HTTP_403_FORBIDDEN,
            detail="Not authorized"
        )
    
    if album.status != "published":
        raise HTTPException(
            status_code=status.HTTP_400_BAD_REQUEST,
            detail="Only published albums can be unpublished"
        )
    
    # 상태 변경
    album.status = "draft"
    album.is_locked = False
    
    db.add(album)
    await db.commit()
    await db.refresh(album)
    
    return _ok(AlbumResponse.model_validate(album).model_dump())
```

const handleUnpublish = async () => {
  if (!album) return
  
  if (!confirm('발행을 취소하시겠습니까?\n앨범을 다시 수정할 수 있게 됩니다.')) {
    return
  }

  try {
    await api.post(`/albums/${albumId}/unpublish`)

    console.error('Unpublish error:', error)
    alert(`발행 취소 실패: ${error.response?.data?.detail || error.message}`)
  } finally {
    setIsUnpublishing(false)
  }
}

## ⚠️ 주의사항

1. **발행 취소 후 곡 삭제 방지**
   - draft 상태에서만 곡 삭제 가능
   - published 상태에서는 제목만 수정 가능

2. **커버 변경 방지**
   - published 상태에서 커버 변경 불가
   - 발행 취소 후 변경 가능

3. **다운로드 링크 무효화**
   - 발행 취소 시 기존 다운로드 링크 무효화 고려
   - 또는 유효한 상태만 다운로드 허용
