## 변경 사항

### 백엔드 수정 (backend/app/services/ai_music.py)

#### Before (234-255줄)
```python
except Exception as e:
    logger.warning(f"Suno API error: {e}. Using mock task_id.")
    return str(uuid.uuid4())
```

#### After
```python
except httpx.HTTPStatusError as e:
    error_detail = f"HTTP {e.response.status_code}"
    try:
        error_json = e.response.json()
        if "error" in error_json:
            error_detail = f"{error_detail}: {error_json['error']}"
        elif "message" in error_json:
            error_detail = f"{error_detail}: {error_json['message']}"
    except:
        error_detail = f"{error_detail}: {e.response.text[:200]}"
    logger.error(f"Suno API error: {error_detail}")
    raise Exception(error_detail) from e
except Exception as e:
    logger.error(f"Suno API error: {str(e)}")
    raise
```

**개선 사항**:
- HTTP 상태 코드와 응답 메시지를 파싱해서 상세 오류 정보 제공
- 예외를 발생시켜 `_run_suno_generation`에서 처리하도록 함
- 오류 메시지가 DB의 `error_message` 필드에 저장됨

- 사용자가 정확한 오류 원인을 알 수 있음

## 기술 상세

### 에러 흐름

1. **API 호출 실패** → Suno API returns 4xx/5xx
2. **예외 발생** → `_call_suno_generate` 에서 예외 발생
3. **오류 저장** → `_run_suno_generation` catch 블록에서 `error_message` DB 저장
4. **상태 업데이트** → Track status = "failed", error_message 저장
5. **프론트엔드 표시** → `music/page.tsx`의 TrackItem에서 error_message 표시

### DB 저장 구조

```
Track.status = "failed"
Track.error_message = "HTTP 400: Invalid API key" 또는 유사한 상세 오류
```

## 결과

사용자는 이제:
1. 생성이 실패하면 정확한 오류 메시지를 볼 수 있음
2. 예: "HTTP 401: Unauthorized" → API 키 문제
3. 예: "HTTP 429: Too Many Requests" → 요청 제한 초과
4. 예: "Connection timeout" → 네트워크 문제
