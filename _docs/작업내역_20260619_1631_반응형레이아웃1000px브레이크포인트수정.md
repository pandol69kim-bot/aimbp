## 작업 내용

앨범 상세 페이지의 트랙 목록에서 1000px 근처 브레이크포인트에서 발생하는 레이아웃 이상 현상을 진단하고 수정했습니다.

## 문제 원인

**파일**: `frontend/src/app/(dashboard)/albums/[id]/page.tsx`

트랙 목록 항목의 구조:
- `flex items-center gap-4` - 가로 배치 고정
- `w-48 shrink-0` - AudioPlayer 너비 고정 (192px)
- 모바일/태블릿에서 공간 부족으로 레이아웃이 비정상적으로 배치됨

## 해결 방법

### 변경 전
```tsx
<div className="flex items-center gap-4 ...">
  <GripVertical ... />
  <span className="w-6 ..." />
  <div className="flex-1 min-w-0">...</div>
  <div className="w-48 shrink-0">  {/* 고정 너비 문제 */}
    <AudioPlayer ... />
  </div>
  <StatusBadge ... />
  <button ...>
```

### 변경 후
```tsx
<div className="flex flex-col sm:flex-row sm:items-center gap-3 sm:gap-4 ...">
  <div className="flex items-center gap-3 sm:gap-4 flex-1 min-w-0">
    <GripVertical className="shrink-0" />
    <span className="w-6 shrink-0" />
    <div className="flex-1 min-w-0">...</div>
  </div>
  <div className="flex items-center gap-2 sm:gap-3 sm:ml-auto">
    <div className="w-32 sm:w-40 md:w-48 shrink-0">  {/* 반응형 너비 */}
      <AudioPlayer ... />
    </div>
    <StatusBadge ... />
    <button className="shrink-0" ... />
  </div>
</div>
```

## 주요 변경 사항

| 항목 | 변경 전 | 변경 후 |
|------|--------|--------|
| **레이아웃** | `flex items-center` (가로 고정) | `flex flex-col sm:flex-row` (반응형) |
| **모바일 (< 640px)** | 한 줄 (오버플로우) | 여러 줄 (자연스러운 스택) |
| **태블릿/데스크톱** | `sm:flex-row` 적용 | `sm:flex-row sm:items-center` 적용 |
| **AudioPlayer 너비** | `w-48` (고정) | `w-32 sm:w-40 md:w-48` (반응형) |
| **Gap 조정** | `gap-4` | `gap-3 sm:gap-4` (모바일에서 간격 감소) |
| **우측 정렬** | 없음 | `sm:ml-auto` (태블릿 이상에서 오른쪽 정렬) |

## 반응형 브레이크포인트

```
모바일 (< 640px)
├─ 레이아웃: 세로 스택
├─ AudioPlayer: w-32
└─ Gap: 3

태블릿 (640px ~ 767px)
├─ 레이아웃: 가로 배치
├─ AudioPlayer: w-40
└─ Gap: 4

데스크톱 (768px 이상)
├─ 레이아웃: 가로 배치
├─ AudioPlayer: w-48 (원래대로)
└─ Gap: 4, ml-auto (우측 정렬)
```
