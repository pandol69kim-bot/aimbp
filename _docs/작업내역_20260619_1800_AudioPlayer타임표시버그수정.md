## 작업 내용

앨범 페이지 AudioPlayer에서 재생 시간(duration/currentTime)이 표시되지 않는 버그 수정.

## 버그 원인 (2가지)

### 버그 1 — useEffect 의존성 누락 (핵심)
```tsx
// 수정 전: deps [] → 마운트 시 1회만 실행
useEffect(() => {
  const audio = audioRef.current
  if (!audio) return  // ← src 없으면 audio 엘리먼트 없어서 null, 리스너 등록 안 됨
  ...
}, [])  // ← src 변경 감지 못함
```
- 컴포넌트 마운트 시 `src`가 없으면 `<audio>` 엘리먼트가 `{src && <audio>}` 조건으로 렌더링되지 않음
- `audioRef.current === null` → 이벤트 리스너 **한 번도 등록 안 됨**
- 이후 데이터 로드돼서 `src`가 생겨도 `useEffect` 재실행 안 함

### 버그 2 — src 변경 시 audio.load() 미호출
- `src` prop이 바뀌어도 브라우저 audio 엘리먼트가 자동 리로드되지 않음
- `durationchange` 이벤트가 발생하지 않아 duration이 0으로 유지됨

## 변경된 파일

### `frontend/src/components/audio/AudioPlayer.tsx`

1. **`<audio>` 엘리먼트 항상 렌더링** (조건부 → 무조건)
   ```tsx
   // 수정 전
   {src && <audio ref={audioRef} src={src} preload="metadata" />}
   // 수정 후
   <audio ref={audioRef} src={src} preload="metadata" />
   ```

2. **`useEffect` 의존성을 `[src]`로 변경** + `audio.load()` 호출
   ```tsx
   useEffect(() => {
     const audio = audioRef.current
     if (!audio) return
     // src 변경 시 상태 초기화
     setCurrentTime(0)
     setDuration(0)
     setIsPlaying(false)
     audio.load()  // 브라우저에 새 파일 로드 지시
     // 이벤트 리스너 등록 ...
   }, [src])  // src가 바뀔 때마다 재실행
   ```

3. **`loadedmetadata` 이벤트 추가** — `durationchange`보다 더 안정적으로 duration 감지
   ```tsx
   audio.addEventListener('loadedmetadata', handleDurationChange)
   ```

4. **duration NaN/Infinity 방어**
   ```tsx
   if (audio.duration && isFinite(audio.duration)) {
     setDuration(audio.duration)
   }
   ```

## 완료 항목

- ✅ 버그 원인 2가지 파악
- ✅ `<audio>` 조건부 렌더링 제거
- ✅ useEffect deps `[]` → `[src]` 변경
- ✅ `audio.load()` 호출 추가
- ✅ `loadedmetadata` 이벤트 리스너 추가
- ✅ duration 무결성 체크 추가
