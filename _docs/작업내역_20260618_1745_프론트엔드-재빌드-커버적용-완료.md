## 작업 내용

앨범 커버 적용 시 `AxiosError: Network Error` 재발 → 프론트엔드 Docker 이미지 재빌드로 해결.

## 원인

이전 세션(`_1730` 작업)에서 `frontend/src/app/(dashboard)/albums/[id]/page.tsx` 를 수정했으나,
"Next.js dev 서버 자동 반영"으로 기록했던 것이 잘못됨.
**이 프로젝트는 Windows USB 경로(공백 포함) 문제로 Docker 볼륨 마운트가 실패하여,
프론트엔드 코드도 반드시 이미지 재빌드가 필요함.**

## 실행 커맨드

```bash
docker-compose build --no-cache frontend
docker-compose up -d frontend
```

## 배포 이후 동작 확인

- `docker logs aimbp-frontend` — TypeScript 에러 없음, `✓ Ready in 8s`

## 최종 커버 적용 흐름

1. "생성된 커버 가져오기" 다이얼로그 → 이미지 클릭 → ring 표시 + `cover_id` 저장
2. "앨범에 적용" → `POST /api/v1/albums/{id}/cover { cover_id }` (소형 payload)
3. 백엔드: base64 디코딩 → `/app/uploads/covers/cover_{id}_{size}.png` 파일 저장
4. `cover.image_url` + `album.cover_url` = 실제 파일 URL
5. 쿼리 무효화 → 헤더 커버 즉시 반영
