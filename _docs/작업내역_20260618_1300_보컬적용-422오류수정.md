## 작업 내용

보컬 적용 시 발생하는 `AxiosError: 422 Unprocessable Entity` 오류를 수정했습니다.

## 원인

프론트엔드가 `vocal_id` 필드명으로 요청을 보냈지만,  
백엔드 `VocalCreate` 스키마는 `library_id` 필드를 요구함.

```
프론트엔드 (구버전) → { track_id, vocal_id, language }   ← 필드명 불일치
백엔드 VocalCreate  → { track_id, library_id, language } ← Pydantic 422 반환
```

`git diff`로 확인: `vocal/page.tsx`에 수정이 working tree에는 있었으나 Docker 이미지 미반영 상태였음.

## 수정된 파일

### `frontend/src/app/(dashboard)/vocal/page.tsx`
- `mutationFn` 타입: `vocal_id` → `library_id`
- `mutateAsync` 호출: `vocal_id: selectedVocalId` → `library_id: selectedVocalId`
- 결과 목록 매칭: `vt.vocal_id` → `vt.library_id`

### `frontend/src/types/index.ts`
- `VocalTrack.vocal_id` → `VocalTrack.library_id`
- `VocalGenerateRequest.vocal_id` → `VocalGenerateRequest.library_id`

## 배포

```bash
docker-compose build --no-cache frontend
docker-compose up -d frontend
```

## 검증

직접 API 테스트로 백엔드 정상 확인:
```
POST /api/v1/vocal/generate
{ track_id, library_id, language: "korean" }
→ HTTP 201 ✅
```
