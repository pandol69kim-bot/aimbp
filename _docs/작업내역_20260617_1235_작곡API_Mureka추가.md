# 작업내역: 작곡 API Mureka 서비스 추가

**날짜:** 2026-06-17 12:35  
**작업자:** Claude Code

## 작업 내용

작곡 API(`POST /api/v1/music/generate`)에 Mureka AI 음악 생성 서비스를 추가했다.
API 키가 설정된 경우 실제 Mureka REST API를 호출하고, 키가 없으면 기존 mock 동작을 유지한다.

## 변경된 파일

- `backend/app/core/config.py`: `MUREKA_API_KEY`, `MUREKA_API_BASE` 설정 및 `has_mureka` 프로퍼티 추가
- `backend/app/services/ai_music.py`: Mureka API 호출·폴링 로직, `VALID_AI_SERVICES` 상수, `call_mureka_api()` / `poll_music_status()` 업데이트
- `backend/app/api/music.py`: `ai_service` 유효성 검사 추가 (`suno` / `mureka` / `udio`)
- `.env`: `MUREKA_API_KEY=your-mureka-api-key` 항목 추가

## 주요 결정 사항

- **실키 없으면 mock fallback**: `has_mureka`가 false이면 5초 지연 후 mock MP3 URL을 반환하므로 키 없이도 동작한다.
- **Mureka API 엔드포인트**: `POST /v1/song/generate` → task_id 반환, `GET /v1/song/{task_id}` 폴링 (최대 24회 × 5초 = 2분)
- **유효성 검사**: `VALID_AI_SERVICES = {"suno", "mureka", "udio"}` — 범위 밖 값은 HTTP 400 반환

## 테스트 결과

| 케이스 | 결과 |
|--------|------|
| `ai_service: "mureka"` 생성 | ✅ processing 상태로 즉시 반환 |
| 7초 후 상태 폴링 | ✅ completed + file_url |
| `ai_service: "invalid_service"` | ✅ HTTP 400 — `ai_service must be one of: mureka, suno, udio` |
