# 작업내역: 메뉴 클릭 오류 해결 - API 응답 형식 통일

**날짜:** 2026-06-17 14:40  
**작업자:** Claude Code

## 문제

메뉴 클릭 시 다른 페이지로 이동하면 TypeError 발생:
- 작사(/lyrics), 작곡(/music), 편곡(/arrangement) 등
- 원인: 페이지별로 API 응답 형식을 다르게 처리

## 근본 원인

각 페이지에서 useQuery 또는 수동 API 호출 시:
```typescript
// ❌ 부정확한 처리
const response = await api.get('/arrangement')
return response.data  // { success, data: [...], error }
```

필요한 배열/객체는 `response.data.data` 에 있음

## 해결 방법

### 1. API 인터셉터 수정 (근본 해결)

`frontend/src/lib/api.ts`:
```typescript
api.interceptors.response.use(
  (response) => {
    // API 응답 형식 자동 변환
    if (response.data?.success && 'data' in response.data && 'error' in response.data) {
      return {
        ...response,
        data: response.data.data,  // ← 자동으로 data 필드 추출
      }
    }
    return response
  },
  // ... 에러 처리
)
```

### 2. 훅 업데이트

인터셉터가 이미 변환하므로, 훅은 response.data를 직접 사용:

```typescript
// useTrackList
const response = await api.get('/music')
return response.data || []  // ← 인터셉터가 이미 처리됨

// useLyricsList
const response = await api.get('/lyrics')
return response.data || []

// useAlbumList
const response = await api.get('/albums')
return response.data || []
```

## 장점

- ✅ **중앙 집중식**: 모든 API 응답 형식을 한 곳에서 관리
- ✅ **자동화**: 새 페이지/훅에서 수동으로 처리할 필요 없음
- ✅ **확장성**: 향후 응답 형식 변경 시 인터셉터만 수정

## 수정된 파일

1. `frontend/src/lib/api.ts` - 응답 인터셉터 추가
2. `frontend/src/hooks/useMusic.ts` - useTrackList 수정
3. `frontend/src/hooks/useLyrics.ts` - useLyricsList 수정
4. `frontend/src/hooks/useAlbums.ts` - useAlbumList 수정

## 추가 이점

인터셉터의 자동 변환으로 인해:
- 모든 페이지의 useQuery가 배열/객체를 올바르게 받음
- 페이지별 useQuery에서 `res.data`만 반환하면 됨
- 향후 새 페이지 추가 시 특별한 처리 불필요
