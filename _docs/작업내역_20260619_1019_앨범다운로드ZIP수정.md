## 작업 내용

앨범 다운로드 ZIP 파일에 트랙 파일이 포함되지 않는 버그 수정.

## 문제 원인

`backend/app/api/albums.py`의 `download_album` 엔드포인트에서 트랙과 커버 파일을 ZIP에 추가할 때 `file_url.startswith("/app/uploads")` 조건만 체크하고 있었음. 그러나 실제 트랙의 `file_url`은 Suno AI 외부 URL (`https://cdn1.suno.ai/...mp3`) 형태로 저장되어 있어 ZIP에 아무것도 포함되지 않는 빈 파일이 다운로드됨.

## 변경된 파일

- `backend/app/api/albums.py`:
  - `import httpx` 추가
  - `_fetch_bytes()` 내부 헬퍼 함수 추가: 로컬 파일(`/app/uploads`)과 외부 URL(`http://`, `https://`) 모두 지원
  - 커버와 트랙 파일 모두 `_fetch_bytes()`로 통일
  - 앨범 제목, 트랙 제목에서 `/`, `\` 문자 이스케이프 처리 추가

## 주요 결정 사항

- `httpx`는 이미 `requirements.txt`에 포함되어 있어 별도 설치 불필요
- 외부 URL fetch 실패 시 해당 파일만 건너뛰고 나머지는 정상 포함 (경고 로그)
- ZIP 내부 구조: `앨범제목/cover.png`, `앨범제목/01. 트랙제목.mp3`
