## 작업 내용

AudioPlayer에서 MP3 파일 로드 실패 시 "재생 불가" 표시 추가.

## 변경된 파일

### `frontend/src/components/audio/AudioPlayer.tsx`

1. **`hasError` 상태 추가**
   ```tsx
   const [hasError, setHasError] = useState(false)
   ```

2. **`error` 이벤트 리스너 추가** — src 로드 실패 시 감지
   ```tsx
   const handleError = () => { setHasError(true); setIsLoading(false); setIsPlaying(false) }
   audio.addEventListener('error', handleError)
   ```
   - src 변경 시 `setHasError(false)` 초기화도 포함

3. **`renderErrorState()` UI 추가** — 오렌지색 경고 아이콘 + "재생 불가" 텍스트
   ```tsx
   const renderErrorState = () => (
     <div className="flex items-center gap-2 text-orange-400" title="파일을 불러올 수 없습니다.">
       <AlertCircle className="h-4 w-4 shrink-0" />
       <span className="text-xs">재생 불가</span>
     </div>
   )
   ```

4. **compact 모드 / full 모드 모두 적용**
   - 에러 시 재생 버튼 비활성화
   - `hasError` → `renderErrorState()` 우선 표시

## 상태별 표시

| 상태 | 표시 |
|------|------|
| processing / pending | 🟡 `생성 중...` (스피너) |
| failed | 🔴 `생성 실패` |
| src 없음 | ⚫ `파일 없음` |
| 로드 에러 (만료 URL 등) | 🟠 `재생 불가` (AlertCircle 아이콘) |
| 정상 | 재생 컨트롤 표시 |
