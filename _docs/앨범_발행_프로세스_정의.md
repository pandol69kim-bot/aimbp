# 앨범 발행(Publish) 프로세스 정의

## 개요

앨범 발행은 작성 중인 앨범을 최종 완성하여 공개하거나, 공개된 앨범을 다시 비공개 상태로 변경하는 프로세스입니다.

---

## 1. 상태 정의

### 앨범 상태

| 상태 | 설명 | 특징 |
|------|------|------|
| **draft** | 작성 중 (기본값) | - 앨범 수정 가능 |
|  |  | - 트랙 추가/제거 가능 |
|  |  | - 커버 변경 가능 |
|  |  | - 사용자만 볼 수 있음 |
|  |  | - 다운로드 불가 |
| **published** | 발행됨 (공개) | - 앨범 정보 수정 불가 |
|  |  | - 트랙 추가/제거 불가 |
|  |  | - 커버 변경 불가 |
|  |  | - 모두가 볼 수 있음 |
|  |  | - 다운로드 가능 |

---

## 2. 앨범 발행 전 체크리스트

발행하기 전에 다음 항목들을 확인해야 합니다.

### 필수 항목 (MUST)
- [ ] 앨범 제목 입력 완료
- [ ] 최소 1개 이상의 완료된 트랙 추가
- [ ] 트랙 순서 정렬 완료
- [ ] 앨범 커버 이미지 설정 (선택사항이지만 권장)

### 권장 항목 (SHOULD)
- [ ] 앨범 설명 작성
- [ ] 각 트랙이 마스터링되어 있음
- [ ] 앨범 커버가 고품질 이미지
- [ ] 트랙 순서가 논리적으로 배열됨

### 정보 항목 (INFO)
- 발행 후에는 수정이 어려움
- 발행 후 비공개로 전환 가능
- 비공개 상태로 변경하면 다운로드 링크는 유효하지 않음

## 3. 발행 프로세스

### 단계 1: 앨범 편집 화면 접근
```
앨범 목록 → 앨범 클릭 → 앨범 상세 페이지 진입
```

### 단계 2: 발행 전 준비
```
1. 앨범 정보 확인
   - 제목
   - 설명
   - 트랙 수

2. 트랙 목록 확인
   - 트랙 개수
   - 트랙 순서
   - 트랙 상태 (완료된 상태인지 확인)

3. 커버 이미지 확인
   - 커버가 설정되어 있는지 확인
   - 필요시 "생성된 커버 가져오기" 클릭하여 변경
```

### 단계 3: 발행하기 버튼 클릭
```
상단 우측 "발행하기" 버튼 클릭
```

### 단계 4: 상태 변경 처리
```
클라이언트 (Frontend)
└─ 발행 버튼 클릭
   └─ PATCH /albums/{album_id} 요청
      ├─ 요청 본문: { "status": "published" }
      └─ 응답: 업데이트된 앨범 정보

서버 (Backend)
└─ PATCH /albums/{album_id} 수신
   ├─ 앨범 소유권 검증
   ├─ status 필드 업데이트
   ├─ updated_at 필드 자동 업데이트
   ├─ DB 저장
   └─ 업데이트된 앨범 응답
```

### 단계 5: UI 반영
```
- 버튼 텍스트 변경: "발행하기" → "비공개로 전환"
- 상태 배지 업데이트: draft → published
- 푸시 알림 또는 토스트 메시지 표시
```

## 5. 비공개 전환 프로세스

### 개념
발행된 앨범을 다시 작성 중 상태로 되돌리는 프로세스

### 언제 필요한가?
- 트랙을 추가해야 할 때
- 트랙을 제거해야 할 때
- 커버 이미지를 변경해야 할 때
- 앨범 정보를 수정해야 할 때

### 프로세스
```
발행된 앨범 상세 페이지
└─ 우측 상단 "비공개로 전환" 버튼 클릭
   └─ PATCH /albums/{album_id} 요청
      ├─ 요청 본문: { "status": "draft" }
      └─ 응답: 업데이트된 앨범 정보

결과:
- 상태가 draft로 변경됨
- 앨범 수정 가능
- 공개 링크 비활성화
```

## 7. 데이터베이스 변화

### Album 테이블 업데이트

```sql
-- 발행하기
UPDATE albums 
SET status = 'published', 
    updated_at = NOW() 
WHERE id = '{album_id}' 
  AND user_id = '{current_user_id}';

-- 비공개로 전환
UPDATE albums 
SET status = 'draft', 
    updated_at = NOW() 
WHERE id = '{album_id}' 
  AND user_id = '{current_user_id}';
```

### 변경되는 필드
| 필드 | 변경 전 | 변경 후 | 설명 |
|------|--------|--------|------|
| **status** | "draft" | "published" | 앨범 상태 |
| **updated_at** | 이전 시간 | 현재 시간 | 마지막 수정 시간 |

### 백엔드 (FastAPI)

```python
# api/albums.py
@router.patch("/{album_id}")
async def update_album(
    album_id: UUID,
    payload: AlbumUpdate,
    current_user: User = Depends(get_current_user),
    db: AsyncSession = Depends(get_db),
):
    # 1. 앨범 조회
    result = await db.execute(
        select(Album).where(
            Album.id == album_id, 
            Album.user_id == current_user.id
        )
    )
    album = result.scalar_one_or_none()
    if not album:
        raise HTTPException(status_code=404, detail="Album not found")
    
    # 2. 상태 업데이트
    if payload.status is not None:
        album.status = payload.status
    
    # updated_at은 자동으로 현재 시간으로 설정됨
    
    # 3. DB 저장
    await db.flush()
    await db.refresh(album)
    
    # 4. 응답
    return _ok(await _build_album_response(album, db))
```

## 11. 발행 전 검증 로직

### 필수 조건
```python
# 검증 함수
def validate_album_for_publish(album: Album) -> tuple[bool, str]:
    """
    앨범 발행 가능 여부 검증
    
    Returns:
        (is_valid, error_message)
    """
    # 1. 제목 확인
    if not album.title or not album.title.strip():
        return False, "앨범 제목을 입력하세요"
    
    # 2. 트랙 확인
    if not album.tracks or len(album.tracks) == 0:
        return False, "최소 1개 이상의 트랙을 추가하세요"
    
    # 3. 모든 트랙이 완료되었는지 확인
    for track in album.tracks:
        if track.status != 'completed':
            return False, f"모든 트랙이 완료되어야 합니다. 미완료: {track.title}"
    
    # 검증 통과
    return True, ""
```

### 사용 예
```python
is_valid, error_msg = validate_album_for_publish(album)
if not is_valid:
    raise HTTPException(status_code=400, detail=error_msg)
```

## 12. 버전 관리 고려사항

### 향후 확장 가능한 상태
```python
# 현재 상태: draft, published
# 향후 추가 가능한 상태:
# - archived: 보관됨 (읽기 전용, 검색 제외)
# - deleted: 삭제됨 (논리적 삭제, 데이터 보존)
# - scheduled: 예약됨 (특정 날짜에 자동 발행)
```

### 상태 전환 규칙
```
draft
  ├─→ published (발행)
  ├─→ archived (보관)
  └─→ deleted (삭제)

published
  ├─→ draft (비공개 전환)
  ├─→ archived (보관)
  └─→ deleted (삭제)

archived
  ├─→ draft (복원)
  └─→ deleted (삭제)

deleted (최종 상태, 복원 불가)
```

## 19. 테스트 케이스

### 단위 테스트
```python
def test_publish_album():
    # 1. draft 상태의 앨범 생성
    album = create_album(status="draft")
    
    # 2. 발행
    result = update_album(album.id, status="published")
    
    # 3. 상태 확인
    assert result.status == "published"
    assert result.updated_at > album.updated_at

def test_unpublish_album():
    # 1. published 상태의 앨범
    album = create_album(status="published")
    
    # 2. 비공개로 전환
    result = update_album(album.id, status="draft")
    
    # 3. 상태 확인
    assert result.status == "draft"

def test_cannot_publish_empty_album():
    # 트랙이 없는 앨범은 발행 불가
    album = create_album(tracks=[])
    
    with pytest.raises(HTTPException):
        update_album(album.id, status="published")
```

### FAQ

**Q: 발행 후 트랙을 추가할 수 없나요?**
A: 현재는 발행 후 수정이 제한됩니다. 필요시 "비공개로 전환"을 클릭하여 다시 편집할 수 있습니다.

**Q: 발행된 앨범을 삭제할 수 있나요?**
A: 비공개로 전환 후 삭제 가능합니다. 발행된 상태에서는 삭제 불가능합니다 (향후 정책 변경 가능).

**Q: 앨범 발행에 비용이 드나요?**
A: 현재로서는 발행 자체에는 비용이 없습니다.

**Q: 몇 개까지 앨범을 만들 수 있나요?**
A: 현재로서는 제한이 없습니다.
