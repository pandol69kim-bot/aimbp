# 편곡(Arrangement) 프로세스 정리

## 개요

편곡은 생성된 트랙에 다양한 악기를 추가하여 음악을 편성하는 프로세스입니다.

---

## 1. 사용자 인터페이스 (Frontend)

### 페이지 위치
- **경로**: `/dashboard/arrangement`
- **파일**: `frontend/src/app/(dashboard)/arrangement/page.tsx`

### UI 구성

#### 왼쪽 패널: 편곡 설정
```
1. 트랙 선택 (드롭다운)
   - 완료된 트랙만 표시 (status === 'completed')
   - 트랙 제목으로 표시

2. 악기 선택 (체크박스 그리드)
   ✓ 드럼
   ✓ 베이스
   ✓ 기타
   ✓ 피아노
   ✓ 스트링
   ✓ 신스
   ✓ 브라스
   ✓ 보컬 코러스

3. 편곡 생성 버튼
   - 트랙 선택 필수
   - 악기 1개 이상 선택 필수
```

#### 오른쪽 패널: 편곡 결과
```
- 편곡 목록 표시
- 각 편곡의:
  - 상태 (Pending / Completed)
  - 생성 시간
  - MP3 재생 기능
  - WAV 다운로드
  - Stems 다운로드
```

## 2. 데이터 모델

### Arrangement (편곡)
```python
# 위치: backend/app/models/arrangement.py

class Arrangement(Base):
    id: UUID (Primary Key)
    track_id: UUID (FK → tracks)  # 대상 트랙
    user_id: UUID (FK → users)     # 소유 사용자
    instruments: JSON              # 선택된 악기 리스트
    wav_url: str | None            # WAV 파일 URL
    mp3_url: str | None            # MP3 파일 URL
    stems: dict | None             # 스템 파일 (악기별)
    status: str                     # pending / completed / failed
    created_at: datetime            # 생성 시간
```

## 4. 백엔드 처리 흐름

### API 엔드포인트 (api/arrangement.py)

#### Step 1: 요청 검증
```python
@router.post("/generate")
async def generate_arrangement(payload, current_user, db):
    # 1. 트랙 소유권 검증
    track = db.query(Track).where(
        Track.id == payload.track_id,
        Track.user_id == current_user.id
    )
    if not track:
        raise 404 "Track not found"
    
    # 2. Arrangement 레코드 생성
    arrangement = Arrangement(
        track_id=payload.track_id,
        user_id=current_user.id,
        instruments=payload.instruments,
        status="pending"
    )
    db.add(arrangement)
    db.flush()
```

#### Step 2: 백그라운드 작업 시작
```python
    # 3. 백그라운드 작업 예약
    start_arrangement_processing(
        arrangement.id,
        background_tasks,
        db_session_factory
    )
    
    # 4. 즉시 응답
    return { "status": "pending", ... }
```

### 백그라운드 처리 (services/arrangement.py)

async def _process_arrangement(arrangement_id, db_session_factory):

## 6. 상태 관리

### Track 상태 (편곡에서 사용 가능한 조건)
```
pending       ❌ 편곡 불가
processing    ❌ 편곡 불가
completed     ✅ 편곡 가능
failed        ❌ 편곡 불가
```

### Arrangement 상태
```
pending    → 처리 대기 중
completed  → 완료, 음악 파일 생성됨
failed     → 처리 실패 (미구현)
```

## 11. 주의 사항

- 현재는 Mock 데이터 반환이므로, 악기 선택이 실제로 반영되지 않습니다
- 편곡 삭제 기능이 미구현되었습니다 (DB 레코드는 남음)
- 에러 처리가 미흡하므로, 실제 AI 엔진 추가 시 개선 필요
