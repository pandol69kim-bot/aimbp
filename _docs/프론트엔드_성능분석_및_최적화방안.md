## 상세 성능 분석

### 1️⃣ 번들 분석

#### 현재 의존성 크기 (예상)
```
next: 300KB
react-dom: 180KB
tailwindcss: 100KB
axios: 15KB
zustand: 3KB
@tanstack/react-query: 50KB
lucide-react: 200KB (아이콘 라이브러리)
────────────────────────
합계: ~850KB (최소값, gzip 전)
```

#### 최적화 후 예상값
```
next: 300KB (고정)
react-dom: 180KB (고정)
optimized bundle: ~250KB
────────────────────────
합계: ~730KB (20% 감소)
```

### 2️⃣ 로딩 워터폴 분석

```
1. HTML 로드 (50ms)
   ↓
2. CSS 로드 (100ms)
   ├─ Tailwind CSS
   └─ Global styles
   ↓
3. JS 로드 및 파싱 (500-800ms)
   ├─ Next.js Runtime
   ├─ React
   ├─ 의존성
   └─ App Bundle
   ↓
4. 하이드레이션 (200-300ms)
   ↓
5. API 요청 시작 (1-3초)
   ├─ 사용자 정보 조회
   ├─ 음악 목록 조회
   ├─ 커버 이미지 조회
   └─ 기타 데이터
   ↓
6. 페이지 렌더링 완료 (3-5초)
```

### 3️⃣ 페이지별 성능

#### 대시보드 (/dashboard)
```
⏱️ 초기 로드 시간: 3-4초
├─ 느린 이유:
│  ├─ 4개의 API 동시 요청
│  ├─ 이미지 100+ 개 로드
│  └─ 복잡한 상태 관리
├─ 개선 후: 1-2초
```

#### 음악 생성 (/music)
```
⏱️ 초기 로드 시간: 2-3초
├─ 느린 이유:
│  ├─ 큰 form 렌더링
│  ├─ 실시간 폴링 (5초 간격)

```

#### 커버 생성 (/cover)
```
⏱️ 초기 로드 시간: 2-3초
├─ 느린 이유:
│  ├─ 수백 개의 커버 이미지 로드
│  ├─ 각 이미지 렌더링 (Base64 또는 URL)

├─ 개선 후: 1-2초
```

## 🎯 최적화 방안 (우선순위별)

### Phase 1: 즉시 적용 가능 (1-2일)

#### 1. 이미지 최적화
```typescript
// 현재 (나쁜 예)
// eslint-disable-next-line @next/next/no-img-element
<img src={imageUrl} alt="cover" />

// 개선 (좋은 예)
import Image from 'next/image'

<Image
  src={imageUrl}
  alt="cover"
  width={400}
  height={300}
  loading="lazy"
  quality={75}
/>
```

**효과**: 30-40% 이미지 크기 감소

#### 2. 동적 임포트 적용
```typescript
// 현재
import { CoverPage } from '@/app/(dashboard)/cover/page'

// 개선
const CoverPage = dynamic(() => import('@/app/(dashboard)/cover/page'), {
  loading: () => <PageLoader />,
  ssr: false
})
```

**효과**: 초기 번들 크기 15-20% 감소

#### 3. API 응답 캐싱 개선
```typescript
// 현재 (TanStack Query 기본 설정)
const { data: tracks } = useQuery({
  queryKey: ['tracks'],
  queryFn: () => api.get('/music')
})

// 개선 (장시간 캐싱)
const { data: tracks } = useQuery({
  queryKey: ['tracks'],
  queryFn: () => api.get('/music'),
  staleTime: 5 * 60 * 1000,      // 5분
  gcTime: 10 * 60 * 1000,         // 10분 (구 cacheTime)
  refetchOnWindowFocus: false
})
```

**효과**: 반복 방문 시 50% 로드 시간 단축

#### 4. CSS 최적화
```typescript
// tailwind.config.ts에서
export default {
  content: [
    './src/**/*.{js,ts,jsx,tsx}',
  ],
  // 사용하지 않는 색상 제거
  theme: {
    colors: {
      // 필요한 색상만 정의
      primary: {...},
      secondary: {...},
      // 불필요한 색상 제거
    }
  }
}
```

**효과**: CSS 번들 크기 10-15% 감소
