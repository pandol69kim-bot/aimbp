## 작업 내용

Docker 컨테이너의 npm/app 모듈 오류를 해결하고 모든 서비스의 정상 기동을 확인했습니다.

### 문제 분석 및 해결

#### 1. Frontend 컨테이너 오류
**문제**: `npm error path /app/package.json - ENOENT`
**원인**: Windows 경로의 공백(`USB Drive`)으로 인한 Docker 볼륨 마운트 실패
**해결 방법**:
- `docker-compose.yml`에서 frontend를 `image: node:20-alpine` 대신 `build:` 사용
- 상대 경로 마운트를 제거하고 이미지 빌드 방식으로 변경
- 이미지 캐시 제거 후 재구축

#### 2. Backend 컨테이너 오류
**문제**: `ModuleNotFoundError: No module named 'app'`
**원인**: `./backend:/app` 볼륨 마운트가 제대로 작동하지 않아 Dockerfile에서 복사한 파일들이 마운트로 덮어씌워짐
**해결 방법**:
- docker-compose.yml에서 `./backend:/app` 마운트 제거
- `uploads_data:/app/uploads` 마운트만 유지 (파일 저장용)
- 이미지 강제 재구축 (`--no-cache`)

## 변경된 파일

- `docker-compose.yml`: 
  - frontend 서비스: `image:` → `build:` 변경, 마운트 제거
  - backend 서비스: `./backend:/app` 마운트 제거

## 주요 결정 사항

**Windows 경로 문제 해결 방식**:
- Docker Desktop on Windows에서 공백/한글이 포함된 경로의 상대 마운트는 불안정
- 해결책: Dockerfile을 통한 이미지 빌드 방식으로 변경 (더 안정적)
- Trade-off: 개발 시 파일 변경 감지가 불가능하지만, 배포 환경에 더 가까운 구성
