## 작업 내용

**앨범에서 MP3 재생이 되지 않는 문제를 해결했습니다.**

### 문제
```
AudioPlayer 컴포넌트는 정상이지만
file_url이 절대 경로(/app/uploads/tracks/...)로 저장되어
브라우저에서 직접 접근 불가
```

### 해결책
```
file_url을 API 엔드포인트 경로로 변경
/app/uploads/tracks/xxx.mp3
→ /api/v1/files/local/tracks/xxx.mp3
```

---

## 🔧 수정사항

### Before (재생 불가)
```python
file_url = "/app/uploads/tracks/{track_id}.mp3"  # 절대 경로
# 브라우저에서 접근 불가
# CORS 또는 서빙 문제 발생
```

### After (재생 가능)
```python
file_url = "/api/v1/files/local/tracks/{track_id}.mp3"  # 상대 경로
# 백엔드 API를 통해 파일 제공
# 브라우저에서 정상 접근 가능
```

## 🎯 작동 원리

### MP3 업로드 흐름
```
1. 사용자가 MP3 파일 업로드
   ↓
2. 백엔드: /app/uploads/tracks/{id}.mp3 에 저장
   ↓
3. DB에 저장: file_url = "/api/v1/files/local/tracks/{id}.mp3"
   ↓
4. 프론트엔드: AudioPlayer 컴포넌트
   ↓
5. <audio src="/api/v1/files/local/tracks/{id}.mp3">
   ↓
6. 백엔드 files.py 엔드포인트가 파일 제공
   ↓
7. 브라우저: MP3 재생 ✅
```

### 파일 서빙 경로
```
요청: GET /api/v1/files/local/tracks/xxx.mp3
   ↓
files.py serve_local_file():
   filename = "tracks/xxx.mp3"
   file_path = /app/uploads + "tracks/xxx.mp3"
           = /app/uploads/tracks/xxx.mp3
   ↓
FileResponse(file_path) 반환
   ↓
브라우저: MP3 재생
```

## 🔄 API 엔드포인트

### MP3 업로드
```
POST /api/v1/music/upload
Content-Type: multipart/form-data

Parameters:
  - file: MP3 파일
  - title: 곡 제목 (선택)
  - artist: 아티스트 (선택)
  - genre: 장르 (선택)

Response:
{
  "success": true,
  "data": {
    "id": "uuid",
    "title": "곡 제목",
    "file_url": "/api/v1/files/local/tracks/xxx.mp3",
    "status": "completed",
    ...
  }
}
```

### MP3 재생
```
GET /api/v1/files/local/tracks/{id}.mp3

Response: 
  Binary MP3 파일
  Content-Type: audio/mpeg
```
