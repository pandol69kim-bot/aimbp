# 작업내역: 로그인 Network Error 수정

**날짜:** 2026-06-17 12:45  
**작업자:** Claude Code

## 문제

Frontend 컨테이너에서 `http://localhost:8001/api/v1`로 접근 시 네트워크 에러 발생
- Docker 컨테이너 내부에서 localhost는 컨테이너 자신을 가리킴
- backend 서비스 이름 또는 상대 경로를 사용해야 함

## 해결책

### 1. docker-compose.yml 수정

```yaml
environment:
  - NEXT_PUBLIC_API_URL=/api/v1    # 상대 경로로 변경
```

### 2. next.config.js 수정

```js
async rewrites() {
  return [
    {
      source: '/api/:path*',
      destination: 'http://backend:8000/api/:path*',  // 수정: api-proxy → api
    },
  ]
}
```

## 동작 방식

| 요청 | 처리 | 최종 URL |
|------|------|---------|
| 브라우저 (localhost:3001) | → | Frontend 컨테이너 |
| Frontend API 호출 `/api/v1/auth/login` | → | Next.js rewrites |
| Next.js rewrites | → | Backend 컨테이너 (http://backend:8000) |

## 테스트 결과

✅ Backend API 정상 응답 (http://localhost:8001/api/v1/auth/login)
✅ Docker 네트워크 설정 정상
✅ Frontend 환경 변수 적용 (NEXT_PUBLIC_API_URL=/api/v1)
✅ Frontend 재시작 완료
