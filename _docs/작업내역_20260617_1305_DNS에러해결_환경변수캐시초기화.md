# 작업내역: DNS 에러 해결 - 환경 변수 캐시 초기화

**날짜:** 2026-06-17 13:05  
**작업자:** Claude Code

## 에러

```
Failed to load resource: net::ERR_NAME_NOT_RESOLVED
http://backend:8000/api/v1/...
```

## 원인

- NEXT_PUBLIC_API_URL이 `http://backend:8000/api/v1` (컨테이너 내부 URL)로 설정됨
- 브라우저에서 "backend" 도메인을 찾을 수 없음 (localhost가 아님)
- docker-compose restart 시 환경 변수 캐시 미반영

## 해결 방법

### 1. docker-compose.yml 명시

```yaml
environment:
  - NEXT_PUBLIC_API_URL=http://localhost:8001/api/v1
  - NODE_ENV=development
```

### 2. next.config.js 정리

불필요한 rewrites 제거:
```js
// 제거된 코드
async rewrites() {
  return [{
    source: '/api/:path*',
    destination: 'http://backend:8000/api/:path*',
  }]
}
```

### 3. 완전 재시작

```bash
docker-compose down && docker-compose up -d
```

## 최종 상태

✅ NEXT_PUBLIC_API_URL=http://localhost:8001/api/v1
✅ axios baseURL이 올바른 절대 URL 사용
✅ 모든 컨테이너 신규 시작
