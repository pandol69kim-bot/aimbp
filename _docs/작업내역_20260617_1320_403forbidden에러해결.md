# 작업내역: 403 Forbidden 에러 해결

**날짜:** 2026-06-17 13:20  
**작업자:** Claude Code

## 에러

```
Request failed with status code 403
/auth/me
```

## 원인

토큰 저장 순서 문제:
```typescript
// ❌ 잘못된 순서
const userData = await fetchCurrentUser()      // 토큰 없음!
setAuth(userData, response.access_token)       // 토큰 저장 (너무 늦음)
```

API 인터셉터가 localStorage에서 토큰을 읽는데, 아직 저장되지 않음

## 해결책

토큰을 먼저 저장한 후 user 정보 조회:

```typescript
// ✅ 올바른 순서
const token = response.access_token
localStorage.setItem('access_token', token)    // 1. 먼저 저장
const userData = await fetchCurrentUser()      // 2. 그 다음 호출 (토큰 포함)
setAuth(userData, token)                       // 3. 마지막에 상태 업데이트
```

## 수정 파일

- `frontend/src/hooks/useAuth.ts`: login 함수 순서 수정
