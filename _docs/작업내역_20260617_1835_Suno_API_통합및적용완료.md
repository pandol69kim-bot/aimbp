## 통합 내용

### 1. Backend 설정 (config.py)
```python
# Suno API 기본 설정 추가
SUNO_API_KEY: str = ""
SUNO_API_BASE: str = "https://api.suno.ai"

# Suno 가용성 확인 속성
@property
def has_suno(self) -> bool:
    return bool(self.SUNO_API_KEY and self.SUNO_API_KEY != "your-suno-api-key")
```

**상태**: ✅ API 키 로드됨 (`f75ab7ac016f596fd8c50cbadc23a6f2`)

### 2. Backend API 구현 (ai_music.py)

#### 새로 추가된 함수
```python
async def _call_suno_generate(title, genre, mood, lyrics) → str
    - Suno /api/generate POST 요청
    - API 키를 Bearer token으로 사용
    - 응답에서 task_id/clip_id 추출

async def _poll_suno_task(task_id) → dict
    - Suno /api/clips/{task_id} GET 요청
    - 상태 폴링 (max 10분)
    - 완료/실패 감지

async def _run_suno_generation(track_id, db_session_factory, track)
    - Suno API 호출 및 폴링
    - 실패 시 Mock MP3로 폴백
    - DB 상태 자동 업데이트
```

#### 기본 로직 수정
```python
# _simulate_music_generation에서 ai_service 확인
if ai_service == "suno" and settings.has_suno:
    await _run_suno_generation(...)
elif ai_service == "mureka" and settings.has_mureka:
    await _run_mureka_generation(...)
else:
    await _mock_generation(...)
```

## 에러 분석

### Suno API 503 에러
```
HTTP Status: 503 Service Unavailable
원인 분석:
1. Suno API 서버 다운 (가능성 높음)
2. API 키 유효하지 않음
3. API 엔드포인트 변경
4. 요청 형식 오류
```

### 해결 방안
```
1. Mock 폴백 활성화 ✅
   - Suno 실패 시 자동으로 Mock MP3 사용
   - 사용자 경험 손상 없음

2. 향후 개선
   - Suno API 상태 모니터링
   - API 키 재검증
   - 대체 음악 생성 서비스 추가
```
