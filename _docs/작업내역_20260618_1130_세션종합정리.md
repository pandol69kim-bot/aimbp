## 2. Suno API 작동 여부 테스트 및 수정

### 문제 진단

| 단계 | 내용 |
|---|---|
| `https://api.suno.ai` | 전면 503 (Cloudflare 뒤, 서버 자체 다운) |
| API 키 형식 | 32자리 hex — sunoapi.org 발급 키로 확인 |
| 기존 코드 방식 | 폴링 루프 → sunoapi.org는 폴링 엔드포인트 없음 |

### 원인 특정

- `api.suno.ai` 엔드포인트 자체가 서비스 중단 상태
- `.env`의 `SUNO_API_KEY`는 **sunoapi.org**에서 발급받은 키
- sunoapi.org는 **웹훅 전용 API** (폴링 불가)

### sunoapi.org API 스펙 확인 (직접 테스트)

```
엔드포인트: POST https://api.sunoapi.org/api/v1/generate
필수 파라미터: model, customMode, instrumental, callBackUrl
유효 모델: V3_5, V4, V4_5, V4_5ALL, V4_5PLUS, V5, V5_5
응답: {"code": 200, "data": {"taskId": "..."}}
```


### `backend/app/core/config.py`
```diff
- SUNO_API_BASE: str = "https://api.suno.ai"
+ SUNO_API_BASE: str = "https://api.sunoapi.org"
+ BACKEND_CALLBACK_URL: str = "http://localhost:8001"
```

### `backend/app/services/ai_music.py`
- `_call_suno_generate()`: sunoapi.org 형식으로 완전 재작성
  - `customMode`, `instrumental`, `callBackUrl` 파라미터 추가
  - 응답에서 `data.taskId` 추출 방식 변경
- `_run_suno_generation()`: 폴링 루프(120회) 제거 → taskId 저장 후 종료
- `_poll_suno_task()`: 제거 (sunoapi.org에 폴링 엔드포인트 없음)

### `backend/app/api/music.py`
- `POST /api/v1/music/webhook/suno` 엔드포인트 신규 추가
  - sunoapi.org가 생성 완료 시 결과를 POST로 전송
  - `taskId`로 Track 조회 후 상태 업데이트
  - `status: completed` → `file_url` 저장
  - `status: failed` → `error_message` 저장

## 4. 최종 동작 흐름

```
사용자 → 프론트엔드 → POST /api/v1/music/generate
                          ↓
                    Track 생성 (status=processing)
                          ↓
                    sunoapi.org API 호출 → taskId 반환
                          ↓
                    Track.task_id = taskId 저장
                          ↓ (sunoapi.org 서버에서 생성 완료)
                    POST /api/v1/music/webhook/suno ← sunoapi.org 콜백
                          ↓
                    Track 상태 업데이트 (completed + file_url)
                          ↓
                    프론트엔드 폴링으로 상태 확인 → 완료 표시
```

## 6. 로컬 개발 제약 및 해결법

sunoapi.org가 `localhost`에 웹훅 전송 불가 → 트랙이 `processing` 상태로 유지됨

**해결 방법 (ngrok 사용):**
```bash
ngrok http 8001
# .env 수정
BACKEND_CALLBACK_URL=https://xxxx.ngrok-free.app
# 백엔드 재빌드
docker-compose build --no-cache backend && docker-compose up -d backend
```

**프로덕션 배포 시:** `BACKEND_CALLBACK_URL=https://api.yourdomain.com`
