# 마스터링(Mastering) 프로세스 정리

## 개요

마스터링은 각 스트리밍 플랫폼의 표준에 맞게 음악의 음량(LUFS)과 음질을 최적화하는 프로세스입니다.

## 2. 데이터 모델

### Mastering (마스터링 작업)
```python
# 위치: backend/app/models/mastering.py

class Mastering(Base):
    id: UUID (Primary Key)
    track_id: UUID (FK → tracks)      # 대상 트랙
    user_id: UUID (FK → users)        # 소유 사용자
    platform: str                     # 타겟 플랫폼
    input_url: str                    # 입력 음악 파일 URL
    output_url: str | None            # 마스터링된 음악 파일 URL
    output_key: str | None            # S3 파일 키
    lufs: float | None                # 최종 음량 (LUFS)
    status: str                       # pending / completed / failed
    created_at: datetime              # 생성 시간
```

## 3. 플랫폼별 LUFS 표준

| 플랫폼 | LUFS 값 | 설명 |
|--------|---------|------|
| **Spotify** | -14.0 | 스트리밍 서비스 표준 |
| **Apple Music** | -16.0 | 고음질 스트리밍 |
| **YouTube Music** | -14.0 | 영상 플랫폼 표준 |
| **SoundCloud** | -10.0 | 낮은 압축 |
| **TIDAL** | -14.0 | 하이파이 스트리밍 |

### LUFS (Loudness Units relative to Full Scale)란?

```
음향 산업 표준으로, 음악의 지각된 음량을 측정하는 단위입니다.

- 높을수록 큰 음량 (-10 LUFS > -14 LUFS > -16 LUFS)
- 각 플랫폼은 자동 정규화를 하므로 목표값 맞춤이 필수
- 목표값보다 크면 음량이 낮춰지고, 작으면 키워집니다
```

## 5. 백엔드 처리 흐름

### API 엔드포인트 (api/mastering.py)

#### Step 1: 요청 검증
```python
@router.post("/process")
async def process_mastering(payload: MasteringCreate, current_user, db):
    # 1. 플랫폼 검증
    VALID_PLATFORMS = {"spotify", "apple_music", "youtube", "soundcloud", "tidal"}
    if payload.platform not in VALID_PLATFORMS:
        raise 400 "Platform must be one of: ..."
    
    # 2. 트랙 소유권 검증
    track = db.query(Track).where(
        Track.id == payload.track_id,
        Track.user_id == current_user.id
    )
    if not track:
        raise 404 "Track not found"
    
    # 3. 입력 URL 결정
    input_url = payload.input_url or track.file_url or ""
    
    # 4. Mastering 레코드 생성
    mastering = Mastering(
        track_id=payload.track_id,
        user_id=current_user.id,
        platform=payload.platform,
        input_url=input_url,
        status="pending"
    )
    db.add(mastering)
    db.flush()
```

#### Step 2: 백그라운드 작업 시작
```python
    # 5. 백그라운드 작업 예약
    start_mastering_processing(
        mastering.id,
        background_tasks,
        db_session_factory
    )
    
    # 6. 즉시 응답

```

## 7. 상태 관리

### Track 상태 (마스터링 가능한 조건)
```
pending       ❌ 마스터링 불가
processing    ❌ 마스터링 불가
completed     ✅ 마스터링 가능
failed        ❌ 마스터링 불가
```

### Mastering 상태
```
pending    → 처리 대기 중
completed  → 완료, 마스터링 파일 생성됨 (LUFS 값 설정됨)
failed     → 처리 실패 (미구현)
```

- 현재는 Mock 데이터 반환이므로, 실제 마스터링 처리가 되지 않습니다

## 15. LUFS 값이 중요한 이유

```
각 스트리밍 플랫폼은 자동으로 음량을 정규화합니다.

예: Spotify는 모든 곡을 -14 LUFS로 정규화

만약 마스터링이 -14 LUFS보다:
- 크면 (예: -10 LUFS) → Spotify가 음량을 낮춤 → 무거운 느낌
- 작으면 (예: -18 LUFS) → Spotify가 음량을 높임 → 왜곡 가능

따라서 목표값 맞춤이 필수입니다!
```

## 16. API 오류 처리

### 400 Bad Request (플랫폼 오류)
```
원인:
- platform이 지정된 값이 아님

해결:
- platform은 다음 중 하나여야 합니다:
  spotify, apple_music, youtube, soundcloud, tidal
```

### 404 Not Found (트랙 없음)
```
원인:
- 트랙이 없거나 소유권이 없음

해결:
- 완료된 트랙만 선택하세요
```
