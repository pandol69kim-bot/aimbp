# 아파트 풍경22 작곡 완료 처리 문제 진단 및 해결

- **날짜**: 2026-06-19 11:45
- **작업자**: Claude

## 문제 현상

- 작곡 "아파트 풍경22"가 **"processing" 상태에서 멈춤**
- 음악 생성 시작 후 완료되지 않음
- 파일 URL이 없음 (file_url: None)

## 원인 분석

### 1. 데이터베이스 상태 확인

```bash
Track ID: 9939bae3-a263-4c7d-9fce-8442f5e001b1
Status: processing  ❌ 완료되지 않음
Task ID: dc3db218473deadc2db2d189b9999392  ✓ 있음
File URL: None  ❌ 없음
AI Service: suno
Created: 2026-06-19 02:30:33
```

### 2. 근본 원인

**Suno API Webhook 콜백이 도착하지 않음**

음악 생성 프로세스:
1. 프론트엔드에서 `/api/v1/music/generate` 호출
2. 백엔드에서 Suno API에 요청 전송
3. ✓ Task ID 반환받아 DB에 저장 → Status: "processing"
4. ❌ **Webhook으로 결과를 기다림 (여기서 도착 안 함)**
5. ❌ Status가 "completed"로 변경 안 됨

**이유:**
- `.env`의 `BACKEND_CALLBACK_URL=https://resample-scribe-busybody.ngrok-free.dev`
- Ngrok 터널이 더 이상 유효하지 않음 (만료되었거나 연결 끊김)
- Suno API가 콜백을 보낼 주소가 없음

### 방안 2: 폴링 방식 도입 (프로덕션 권장)

Webhook 대신 주기적으로 Suno API에서 작곡 상태를 조회:

```python
# ai_music.py 수정
async def _run_suno_generation_with_polling(track_id, db_session, track):
    """Suno API에 요청 후 주기적으로 상태 조회"""
    task_id = await _call_suno_generate(...)
    
    # 최대 10분 동안 30초마다 조회
    for attempt in range(20):
        await asyncio.sleep(30)
        
        # Suno API에서 상태 조회
        status = await _poll_suno_task(task_id)
        
        if status["state"] == "complete":
            # 완료됨
            db_track.status = "completed"
            db_track.file_url = status["audio_url"]
            await db_session.commit()
            break
        elif status["state"] == "error":
            # 실패
            db_track.status = "failed"
            await db_session.commit()
            break
```
