- **날짜**: 2026-06-18 18:00

## 작업 내용

앨범 커버 이미지 엑박(broken image) 수정.

## 근본 원인

`.env`의 `BACKEND_CALLBACK_URL=https://resample-scribe-busybody.ngrok-free.dev` 값이 커버 파일 URL 생성에도 사용되고 있었음.
ngrok free tier는 `<img src="https://xxx.ngrok-free.dev/...">` 로 이미지를 로드할 때 **브라우저 인터스티셜 경고 HTML 페이지**를 반환 → 이미지 엑박.

## 변경된 파일

### `.env`
- `FILE_BASE_URL=http://localhost:8001` 추가
  (파일 서빙 전용 URL — ngrok가 아닌 직접 포트)

### `backend/app/core/config.py`
- `FILE_BASE_URL: str = "http://localhost:8001"` 설정 추가

### `backend/app/api/albums.py`
- `apply_cover_to_album()`: `BACKEND_CALLBACK_URL` → `FILE_BASE_URL` 사용

### `backend/app/services/cover.py`
- `_save_cover_to_file()`: `BACKEND_CALLBACK_URL` → `FILE_BASE_URL` 사용

### DB 직접 수정 (일회성)
- `covers.image_url`, `albums.cover_url`에 저장된 ngrok URL을 `localhost:8001` URL로 교체
  ```sql
  UPDATE covers SET image_url = REPLACE(image_url, 'https://resample-...ngrok-free.dev', 'http://localhost:8001') ...
  UPDATE albums SET cover_url = REPLACE(cover_url, ...)
  ```

## 역할 분리 (수정 후)

| 설정 | 용도 | 값 |
|---|---|---|
| `BACKEND_CALLBACK_URL` | Suno 웹훅 콜백 URL (ngrok 필요) | `https://resample-scribe-busybody.ngrok-free.dev` |
| `FILE_BASE_URL` | 파일 서빙 URL (직접 포트) | `http://localhost:8001` |

## 배포

```bash
docker-compose build --no-cache backend
docker-compose up -d backend
```
