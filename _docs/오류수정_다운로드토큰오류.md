# 다운로드 실패: Invalid or expired token 오류 수정

- **날짜**: 2026-06-19 16:26
- **오류**: 다운로드 실패: Invalid or expired token
- **원인**: 토큰 처리 방식의 불일치
- **상태**: ✅ 수정 완료

---

## 🐛 문제 분석

### 오류 증상
```
다운로드 실패: Invalid or expired token
```

### 근본 원인
프론트엔드의 다운로드 함수가 **fetch API**를 사용하면서 토큰 처리가 제대로 되지 않음

**Before (문제 있는 코드)**
```typescript
const token = localStorage.getItem('access_token')
const response = await fetch(baseUrl, {
  headers: {
    Authorization: `Bearer ${token}`,
  },
})
```

**문제점:**
1. `localStorage`에서 토큰을 직접 읽기 → 형식 불일치 가능
2. `fetch` API는 API 인터셉터를 사용하지 않음 → 토큰 검증 로직 누락
3. 에러 처리에서 `response.json()` 후 `response.blob()` 호출 불가능 (이미 읽음)

---

## ✅ 해결 방법

### 수정 사항
```typescript
// After (수정된 코드)
const response = await api.get(`/albums/${albumId}/download`, {
  responseType: 'blob',  // blob 형식 지정
})

const blob = response.data
```

### 변경 내용

| 항목 | Before | After |
|------|--------|-------|
| HTTP 클라이언트 | fetch API | axios (api) |
| 토큰 처리 | 수동 (localStorage) | 자동 (인터셉터) |
| 응답 형식 | fetch response | axios response |
| 에러 처리 | response.json() | error.response.data |

## 📝 코드 비교

### Before (문제)
```typescript
const handleDownload = async () => {
  const token = localStorage.getItem('access_token')  // 직접 토큰
  const baseUrl = process.env.NEXT_PUBLIC_API_URL || 'http://localhost:8001/api/v1'
  
  const response = await fetch(`${baseUrl}/albums/${albumId}/download`, {
    headers: {
      Authorization: `Bearer ${token}`,  // 수동 헤더 추가
    },
  })
  
  if (!response.ok) {
    const errJson = await response.json()  // 바디 읽음
    throw new Error(errJson?.detail)
  }
  
  const blob = await response.blob()  // ❌ 이미 읽은 바디 다시 읽기 불가능
}
```

### After (수정)
```typescript
const handleDownload = async () => {
  const response = await api.get(`/albums/${albumId}/download`, {
    responseType: 'blob',  // blob 형식 사전 지정
  })
  
  const blob = response.data  // 직접 blob 액세스
  // ✅ api 라이브러리가 토큰 자동 추가 및 처리
}
```

## 🔧 기술 세부사항

### API 라이브러리 (axios) 사용
```typescript
// lib/api.ts
const api = axios.create({
  baseURL: process.env.NEXT_PUBLIC_API_URL,
})

// 인터셉터가 자동으로 토큰 추가
api.interceptors.request.use((config) => {
  const token = localStorage.getItem('access_token')
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})
```

### Blob 응답 처리
```typescript
// 1. 요청 단계에서 응답 형식 지정
const response = await api.get(url, {
  responseType: 'blob',  // Content-Type: application/octet-stream
})

// 2. 응답에서 직접 blob 액세스
const blob = response.data  // ✅ Blob 객체

// 3. 다운로드 처리
const url = window.URL.createObjectURL(blob)
const link = document.createElement('a')
link.href = url
link.download = 'file.zip'
link.click()
```

## 📚 관련 코드

### API 인터셉터 설정
**파일**: `frontend/src/lib/api.ts`
```typescript
import axios from 'axios'

const api = axios.create({
  baseURL: process.env.NEXT_PUBLIC_API_URL,
})

// 요청 인터셉터 (토큰 자동 추가)
api.interceptors.request.use((config) => {
  const token = localStorage.getItem('access_token')
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})

export default api
```

### 다운로드 함수 (수정)
```typescript
const handleDownload = async () => {
  if (!album) return
  setIsDownloading(true)
  try {
    // API 라이브러리 사용 (토큰 자동 포함)
    const response = await api.get(`/albums/${albumId}/download`, {
      responseType: 'blob',
    })

    const blob = response.data
    const url = window.URL.createObjectURL(blob)
    const link = document.createElement('a')
    link.href = url
    link.setAttribute('download', `${album.title}.zip`)
    document.body.appendChild(link)
    link.click()
    link.parentNode?.removeChild(link)
    window.URL.revokeObjectURL(url)
  } catch (error: any) {
    const errorMsg =
      error.response?.data?.detail ||
      error.message ||
      '알 수 없는 오류'
    alert(`다운로드 실패: ${errorMsg}`)
  } finally {
    setIsDownloading(false)
  }
}
```

## 💡 핵심 교훈

### fetch API vs axios

| 특징 | fetch | axios |
|------|-------|-------|
| 토큰 자동 추가 | ❌ 수동 | ✅ 인터셉터 |
| 응답 처리 | 복잡 | 간단 |
| 에러 처리 | status 확인 필요 | 자동 |
| 기본값 설정 | 불가능 | 가능 |

**결론**: HTTP 요청에는 axios (또는 api 라이브러리) 사용 권장
