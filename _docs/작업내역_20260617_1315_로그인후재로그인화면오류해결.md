# 작업내역: 로그인 후 재로그인 화면 오류 해결

**날짜:** 2026-06-17 13:15  
**작업자:** Claude Code

## 문제

로그인 후 대시보드로 이동하지 않고 다시 로그인 화면으로 돌아감

## 원인 분석

### 1. Backend 응답 부족
API 응답에 `user` 정보가 없음:
```json
{
  "success": true,
  "data": {
    "access_token": "...",
    "refresh_token": "...",
    "token_type": "bearer"
  }
}
```

Frontend 예상: `response.user` → undefined!

### 2. Zustand Hydration 타이밍
Dashboard layout이 persist middleware hydration 완료 전에 인증 확인
→ 항상 `!accessToken || !user` 조건 발동 → 다시 로그인 화면으로 이동

## 해결 방법

### 1. useAuth.ts 수정

로그인 후 `/auth/me` 호출로 user 정보 가져오기:

```typescript
const login = async (data: LoginRequest) => {
  setIsLoading(true)
  setError(null)
  try {
    const response = await loginUser(data)
    const userData = await fetchCurrentUser()  // ← 추가
    setAuth(userData, response.access_token)   // ← user 정보 포함
    router.push('/dashboard')
  } catch (err: unknown) {
    // ... 에러 처리
  }
}
```

### 2. dashboard/layout.tsx 수정

Zustand hydration 상태 추가:

```typescript
const [isHydrated, setIsHydrated] = useState(false)

useEffect(() => {
  setIsHydrated(true)  // 클라이언트에서 hydration 완료
}, [])

useEffect(() => {
  if (!isHydrated) return  // hydration 완료 대기
  if (!accessToken || !user) {
    router.replace('/login')
  }
}, [isHydrated, accessToken, user, router])
```

## 수정 파일

- `frontend/src/hooks/useAuth.ts`: 로그인 후 user 정보 조회
- `frontend/src/app/(dashboard)/layout.tsx`: hydration 상태 관리
