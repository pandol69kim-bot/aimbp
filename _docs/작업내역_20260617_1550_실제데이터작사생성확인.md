# 작업내역: 실제 데이터 작사 생성 확인

**날짜:** 2026-06-17 15:50  
**작업자:** Claude Code

## 문제

작사가 Mock 데이터로 나타남

### 원인 분석
Backend 로그:
```
WARNING - OpenAI API key not set; returning mock lyrics
```

Backend 환경 변수:
```
OPENAI_API_KEY=your-openai-api-key  ❌ (기본값)
```

### 근본 원인
- .env 파일이 있지만 Docker에 제대로 로드되지 않음
- Docker 컨테이너가 기본값을 사용 중

## 해결 방법

### 1. Docker 전체 재구축
```bash
cd /path/to/AIMBP
docker-compose down
docker-compose up -d --build
```

### 2. 환경 변수 확인
Before:
```
OPENAI_API_KEY=your-openai-api-key
```

After:
```
OPENAI_API_KEY=sk-proj-lA57Kf41E8MHql4D...
```

## 테스트 결과

### ✅ 실제 작사 생성
```
요청:
- 제목: 달빛 속의 약속
- 주제: 사랑, 약속, 영원함
- 분위기: 로맨틱하고 설레는
- 장르: K-POP
- 모델: OpenAI

응답:
✅ 한글 가사 생성 (verse, chorus, bridge, hook)
✅ 실제 OpenAI GPT-4o-mini API 호출 확인
```

### 생성된 가사 예시
```
Verse:
언제 같은 달 아래, 꿈을 나누던 그대,
한 번의 별빛처럼, 반짝이는 우리의 맹세.
당신 속에, 내 사랑을 담아,
사라지지 않는 기억으로 남아.

Chorus:
사랑해, 이 시간에, 도망 함께라면,
매일이 특별해, 너를 느끼고 싶어.
이 밤은 우리의 이야기,
영원히 계속될 거야, 다 함께 불러내.
```

## Docker env_file 동작

docker-compose.yml:
```yaml
backend:
  env_file:
    - .env
```

- Docker Compose는 실행 디렉토리의 .env 파일을 자동 로드
- 전체 재구축 시 .env가 제대로 로드됨

## 최종 상태

| 기능 | 상태 | 상세 |
|------|------|------|
| **OpenAI API 키** | ✅ 로드됨 | sk-proj-... |
| **작사 생성** | ✅ 실제 데이터 | GPT-4o-mini |
| **Mock 데이터** | ❌ 사용 안 함 | 제거됨 |

## 향후 주의

- Docker 컨테이너 재시작할 때 `docker-compose up -d --build` 사용
- 또는 `--build` 없이 `docker-compose restart`도 환경변수 유지

## 검증 방법

1. Frontend: http://localhost:3001 → 작사 생성
2. 드롭다운에서 "OpenAI GPT-4o-mini" 선택
3. 제목, 주제, 분위기 등 입력
4. "작사 생성하기" 버튼 클릭
5. 실제 한글 가사 생성 확인

### Backend 로그 확인
```bash
docker logs aimbp-backend | grep -i "OpenAI API key"

# ✅ 성공 (API 키 로드됨)
# ❌ 실패 (기본값)
```
