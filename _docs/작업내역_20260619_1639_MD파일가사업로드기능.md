## 작업 내용

**작사된 MD 파일을 업로드하면 생성 목록에 추가되는 기능을 구현했습니다.**

사용자가 이미 작사한 마크다운 파일을 업로드하면 AI 생성 가사와 함께 "생성 목록"에 표시됩니다.

```python
@router.post("/upload", status_code=status.HTTP_201_CREATED)
```

#### 기능
- ✅ `.md` 파일만 수락
- ✅ 파일 내용 파싱
  - `# Title` → 제목 추출
  - `## Verse` → 가사 섹션 분류
  - `## Chorus` → 후렴 섹션
  - `## Bridge` → 브릿지 섹션
  - `## Hook` → 훅 섹션
- ✅ 데이터베이스에 저장
- ✅ 생성 목록에 표시

#### 드래그 드롭 구현
```typescript
const handleDrag = (e: React.DragEvent) => {
  if (e.type === 'dragenter' || e.type === 'dragover') {
    setIsDragActive(true)
  } else if (e.type === 'dragleave') {
    setIsDragActive(false)
  }
}

const handleDrop = (e: React.DragEvent) => {
  const files = Array.from(e.dataTransfer.files).filter((file) =>
    file.name.toLowerCase().endsWith('.md')
  )
  if (files.length > 0) {
    setUploadFile(files[0])
  }
}
```

## 📝 MD 파일 형식

### 예시 파일 (example.md)
```markdown
# Title: 첫사랑

## Verse
밤하늘 별처럼 반짝이던 너의 눈
처음 만났던 그날의 설렘이 아직도

## Chorus
너를 사랑해 첫사랑
영원히 잊혀지지 않을 너

## Bridge
시간이 흘러도 변하지 않는 마음

## Hook
첫사랑, 너였어
```

### 지원 섹션
- `# Title` - 곡 제목
- `## Verse` - 가사 (여러 절 가능)
- `## Chorus` - 후렴
- `## Bridge` - 브릿지
- `## Hook` - 훅/모티브

## 💡 구현 노트

### 왜 드래그 드롭을 사용했는가?
- 사용자 경험 향상 (직관적)
- MP3 업로드와 일관성
- 빠른 파일 추가

### 왜 MD 형식을 선택했는가?
- 가장 간단한 텍스트 형식
- 섹션 분류가 명확
- 편집이 용이
- 버전 관리 친화적 (git)

### 파싱 알고리즘
```
1. 파일을 줄 단위로 분할
2. 각 줄에 대해:
   - 섹션 헤더 감지 (## Title, ## Verse 등)
   - 콘텐츠 줄 누적
3. 섹션별로 콘텐츠 저장
```
