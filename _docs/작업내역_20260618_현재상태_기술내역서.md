## 📋 프로젝트 개요

### 목적
YouTube 동영상용 AI 작곡 웹 애플리케이션
- 가사 자동 생성 (GPT-4)
- 음악 생성 (Suno AI, Mureka AI, Udio AI)
- 보컬 생성 (TTS)
- 음악 편집 및 다운로드

### 기술 스택

#### Frontend
- **프레임워크**: Next.js 15 (App Router)
- **언어**: TypeScript
- **UI/스타일**: Tailwind CSS + 커스텀 디자인 시스템
- **상태 관리**: React Query (TanStack Query)
- **API**: fetch + TanStack Query mutations

#### Backend
- **프레임워크**: FastAPI (Python)
- **데이터베이스**: PostgreSQL 16
- **ORM**: SQLAlchemy 2.0
- **인증**: JWT (Bearer Token)
- **API 클라이언트**: httpx

#### 인프라
- **컨테이너**: Docker & Docker Compose
- **데이터베이스**: PostgreSQL 16 (docker)
- **로컬 개발**: Docker Compose

## 🐛 최근 해결된 문제 (2026-06-18)

### 1. Docker 컨테이너 오류
**문제**: Windows 경로 공백으로 인한 Docker 볼륨 마운트 실패
```
❌ npm error path /app/package.json - ENOENT
❌ ModuleNotFoundError: No module named 'app'
```

**해결방법**:
- `docker-compose.yml` 수정
- Frontend: 마운트 제거, `build:` 방식 사용
- Backend: `./backend:/app` 마운트 제거
- 이미지 캐시 제거 후 재구축

**결과**: ✅ 모든 컨테이너 정상 기동

### 2. Suno API 오류 처리
**문제**: API 실패 시 사용자에게 오류 메시지가 표시되지 않음
- 백엔드: 예외를 조용히 처리 (로그만 남김)
- 프론트엔드: 일반적인 메시지만 표시

**해결방법**:
- 백엔드: HTTP 상태 코드와 응답 메시지 파싱
- 프론트엔드: 상세 오류 메시지를 박스 형태로 표시
- 데이터베이스: `error_message` 필드에 오류 저장

**결과**: ✅ 사용자가 정확한 오류 메시지 확인 가능

## 📞 문제 해결 가이드

### "API 연결 실패" 에러
```
해결:
1. Backend 서비스 상태 확인: docker-compose logs backend
2. API_URL 환경 변수 확인
3. CORS 설정 확인 (backend/app/main.py)
```

### "인증 실패" 에러
```
해결:
1. JWT_SECRET 환경 변수 설정 확인
2. 토큰 만료 시간 확인
3. localStorage의 토큰 확인 (DevTools > Application)
```

### "Suno API 오류"
```
해결:
1. SUNO_API_KEY 설정 확인
2. API 할당량 확인 (console.log 메시지)
3. 네트워크 연결 확인
```
