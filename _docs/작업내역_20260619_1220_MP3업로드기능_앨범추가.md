## 작업 내용

**기존 Suno AI 음악 생성 방식은 유지하면서**, MP3 파일을 직접 업로드해서 같은 앨범에 추가할 수 있는 기능을 구현했습니다.

## 구현 완료 항목

### 1. ✅ Track 모델 수정
- `artist_name` 필드 추가 (트랙의 아티스트명)
- 모든 트랙이 제목, 아티스트명, 장르 정보를 가지도록 통일

### 2. ✅ MP3 업로드 API 추가
**엔드포인트**: `POST /api/v1/music/upload`

**요청 형식**:
```
Content-Type: multipart/form-data

file: <MP3 파일>
title: "곡 제목" (선택사항 - 파일명 사용)
artist: "아티스트명" (선택사항 - 사용자 닉네임 사용)
genre: "장르" (선택사항)
```

**응답**:
```json
{
  "success": true,
  "data": {
    "id": "track-uuid",
    "title": "Test Track from Upload",
    "artist_name": "Test Artist",
    "genre": "Test Genre",
    "file_url": "/app/uploads/tracks/track-uuid.mp3",
    "status": "completed",
    "ai_service": "upload"
  }
}
```

### 3. ✅ 파일 저장 로직
- MP3 파일을 `/app/uploads/tracks/` 디렉토리에 저장
- 파일명: `{track_id}.mp3`
- 즉시 `status: "completed"`로 설정 (생성 대기 불필요)

### 4. ✅ 트랙 구분
- `ai_service` 필드로 음악 생성 방식 구분
  - `"suno"`: Suno AI로 생성
  - `"mureka"`: Mureka AI로 생성  
  - `"upload"`: 사용자 업로드
- 앨범 다운로드 시 모든 트랙 포함

### 5. ✅ 데이터베이스 마이그레이션
- `artist_name` 컬럼을 `tracks` 테이블에 추가
- 기존 데이터와의 호환성 유지

## 사용 방법

### 1. MP3 파일 업로드

```bash
curl -X POST http://localhost:8001/api/v1/music/upload \
  -H "Authorization: Bearer <token>" \
  -F "file=@path/to/song.mp3" \
  -F "title=My Song" \
  -F "artist=My Artist" \
  -F "genre=Pop"
```

### 2. 업로드된 트랙을 앨범에 추가

```bash
# 먼저 앨범 생성
POST /api/v1/albums
{
  "title": "My Album",
  "description": "Album with uploaded and AI-generated tracks"
}

# 업로드한 트랙 추가
POST /api/v1/albums/{album_id}/tracks
{
  "track_id": "<uploaded_track_id>",
  "order": 1
}

# Suno AI로 생성한 트랙도 같이 추가
POST /api/v1/albums/{album_id}/tracks
{
  "track_id": "<suno_generated_track_id>",
  "order": 2
}
```

### 3. 앨범 발행 및 다운로드

```bash
# 앨범 발행
POST /api/v1/albums/{album_id}/publish

# 앨범 다운로드 (모든 트랙 포함)
GET /api/v1/albums/{album_id}/download
```

## 핵심 특징

### 혼합 트랙 앨범
- **Suno AI 생성 곡** + **사용자 업로드 MP3** 같은 앨범에 포함 가능
- 각 트랙의 출처(`ai_service`) 기록
- 앨범 다운로드 시 모든 트랙 포함 (ZIP 형식)

### 메타데이터
- 각 트랙마다 독립적인 아티스트명 저장
- 앨범과 독립적으로 트랙 정보 관리
- 향후 가사, 작곡가 등 추가 정보 확장 가능

### 호환성
- 기존 Suno AI 방식 완벽하게 유지
- 새로운 업로드 기능과 공존
- 기존 앨범에도 업로드 트랙 추가 가능

## 구현 세부사항

### Track 모델 변경
```python
class Track(Base):
    ...
    artist_name: Mapped[str | None] = mapped_column(String(255), nullable=True)
    ...
    ai_service: Mapped[str | None] = mapped_column(String(50), nullable=True)
    # "suno", "mureka", "upload" 등
```

### MP3 저장 로직
```python
# /app/uploads/tracks/{track_id}.mp3로 저장
uploads_dir = "/app/uploads/tracks"
os.makedirs(uploads_dir, exist_ok=True)

file_path = f"{uploads_dir}/{track_id}.mp3"
with open(file_path, 'wb') as f:
    f.write(file_contents)

# 트랙 상태: 즉시 "completed"
track.status = "completed"
track.file_url = file_path
```
