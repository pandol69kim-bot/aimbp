## 1. GPT 작사 생성 확인

### 테스트 결과
```
POST /api/v1/lyrics/generate
요청: ai_model: "openai"
응답: 한글 가사 (verse, chorus, bridge, hook)
✅ OpenAI GPT-4o-mini 실제 호출 확인됨
```

### 생성된 가사 예시
```
주제: 인공지능과 미래

장르: 힙합
언어: 한국어

Verse:
이른 아침의 빛을 찾아
긴 밤을 걸어왔어
수많은 시간들이 흘러
이제야 알 것 같아

Chorus:
날아올라 끝없는 하늘로
두려움을 뒤로하고
새로운 세상을 향해
나는 날아가
```

## 2. 모델 선택 기능 수정

### 문제
Frontend의 AI 모델 옵션이 Backend 지원 모델과 불일치:
```typescript
// ❌ 잘못된 옵션
const aiModelOptions = [
  { value: 'gpt-4', label: 'GPT-4' },
  { value: 'claude-3', label: 'Claude 3' },
  { value: 'gemini-pro', label: 'Gemini Pro' },
]
```

### 해결
Backend 실제 지원 모델로 수정:
```typescript
// ✅ 올바른 옵션
const aiModelOptions = [
  { value: 'openai', label: 'OpenAI GPT-4o-mini' },
  { value: 'claude', label: 'Anthropic Claude Haiku' },
]
```

### 기본값 변경
```typescript
// 변경 전
const [aiModel, setAiModel] = useState('gpt-4')

// 변경 후
const [aiModel, setAiModel] = useState('openai')
```

## 지원하는 AI 모델

| 모델 | 값 | 상태 | 특징 |
|------|-----|------|------|
| **OpenAI GPT-4o-mini** | `openai` | ✅ | 빠른 생성, 한글 우수 |
| **Anthropic Claude Haiku** | `claude` | ✅ | 깊이있는 생성, 고비용 |

## 테스트 방법

1. http://localhost:3001 접속
2. 작사 메뉴 → 생성 설정
3. AI 모델 드롭다운 확인
   - "OpenAI GPT-4o-mini" (기본)
   - "Anthropic Claude Haiku"
4. 각 모델로 작사 생성 테스트

## API 호출 흐름

```
Frontend (모델 선택)
  ↓
POST /api/v1/lyrics/generate
  {
    title: string,
    prompt_subject: string,
    prompt_mood: string,
    prompt_genre: string,
    prompt_language: string,
    ai_model: "openai" | "claude"  ✅ 수정됨
  }
  ↓
Backend (선택된 모델로 호출)
  - openai → OpenAI API 호출
  - claude → Anthropic API 호출
  ↓
Response
  {
    id: UUID,
    title: string,
    verse: string,
    chorus: string,
    bridge: string,
    hook: string,
    ai_model: string  ✅ 확인 가능
  }
```
