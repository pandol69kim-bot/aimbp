## 작업 내용

**MP3 파일을 드래그 드롭으로 여러 개 동시에 업로드할 수 있는 기능을 구현했습니다.**

기존의 단일 파일 선택 방식에서 멀티 파일 드래그 드롭 방식으로 개선했습니다.

## 구현 완료 항목

- 파일을 드래그 드롭할 수 있는 영역 추가
- 마우스 오버 시 하이라이트 (primary-500 색상)
- 클릭으로도 파일 선택 가능

- 파일 목록에 표시
- 개별 파일 제거 기능

### 3. ✅ 진행 상태 표시
- 파일별 업로드 진행도 (프로그레스 바)
- 업로드 완료 표시 (✓)
- 업로드 실패 표시 (✗)

- 여러 파일 동시 업로드
- 순차 처리 (하나씩 업로드)
- 완료 메시지 (n개 추가됨)

## 기술 구현

### 상태 관리
```typescript
const [uploadFiles, setUploadFiles] = useState<File[]>([])
const [isDragActive, setIsDragActive] = useState(false)
const [uploadProgress, setUploadProgress] = useState<{[key: string]: number}>({})
```

### 드래그 드롭 핸들러
```typescript
const handleDrag = (e: React.DragEvent) => {
  e.preventDefault()
  if (e.type === 'dragenter' || e.type === 'dragover') {
    setIsDragActive(true)
  } else if (e.type === 'dragleave') {
    setIsDragActive(false)
  }
}

const handleDrop = (e: React.DragEvent) => {
  e.preventDefault()
  const files = Array.from(e.dataTransfer.files).filter((file) =>
    file.name.toLowerCase().endsWith('.mp3')
  )
  setUploadFiles((prev) => [...prev, ...files])
}
```


## 사용 방법

### 1. 앨범 열기
```
앨범 상세 페이지 → [트랙 추가] 버튼
```

### 2. MP3 업로드 탭 선택
```
[기존 트랙] | [MP3 업로드] ← 클릭
```

### 3. 파일 선택 (2가지 방법)

**방법 1: 드래그 드롭**
```
📁 MP3를 드래그하거나 클릭
↓
파일을 이 영역에 드래그
↓
자동으로 목록에 추가
```

**방법 2: 클릭 선택**
```
📁 영역 클릭
↓
파일 선택 대화상자 열림
↓
여러 파일 선택 (Ctrl+클릭)
↓
확인
```

### 4. 파일 목록 확인
```
선택된 파일 (3개)
├── song1.mp3    [✓ 완료] [제거]
├── song2.mp3    [진행 중] 
└── song3.mp3    [대기 중] [제거]
```

### 5. 업로드 시작
```
[3개 업로드] 버튼 클릭
↓
순차적으로 업로드
↓
진행도 표시
↓
"3개의 곡이 앨범에 추가되었습니다."
```

### ✅ 드래그 드롭
- MP3 파일을 드래그해서 영역에 놓기
- 여러 파일 한 번에 선택
- 마우스 오버 시 하이라이트

### ✅ 파일 목록 관리
- 선택된 파일 표시

- 개별 파일 제거

### ✅ 진행 상태 추적
- 파일별 진행 상황 표시
- 완료/실패 표시

### ✅ 일괄 처리
- 여러 파일 순차 업로드

- 완료 후 확인 메시지

## 에러 처리

**파일 유효성**
- MP3 파일만 선택 가능
- 다른 형식은 자동 필터링

**업로드 실패**
- 개별 파일 실패해도 다른 파일 계속 업로드
- 실패한 파일 표시 (✗)
- 완료 메시지에 성공한 개수만 표시

**사용자 확인**
- 파일 제거 확인 (즉시 삭제)
- 업로드 완료 메시지

## 성능 고려사항

### 업로드 방식
- **순차 처리**: 한 파일씩 업로드 (서버 부하 고려)
- **병렬 처리 가능**: 필요시 `Promise.all()` 사용

### 메모리 관리
- 파일 객체는 메모리에 임시 보관
- 업로드 후 불필요한 객체 정리

### 네트워크
- 각 파일의 진행도 실시간 업데이트
- 완료 시 쿼리 재검증
