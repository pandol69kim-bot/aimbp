# AIMBP 서버 구동 방법 - 유저 가이드

## 📋 시스템 요구사항

### 필수 설치 프로그램

| 프로그램 | 버전 | 용도 |
|---------|------|------|
| **Docker** | 24.0+ | 컨테이너화된 서비스 |
| **Docker Compose** | 2.0+ | 다중 컨테이너 오케스트레이션 |
| **Git** | 2.30+ | 코드 관리 |
| **Node.js** | 20.x | 프론트엔드 개발 (선택사항) |
| **Python** | 3.11+ | 백엔드 개발 (선택사항) |

### 시스템 사양

```
CPU: 4 코어 이상 권장
RAM: 8GB 이상 권장
저장소: 20GB 이상 (데이터베이스용)
네트워크: 인터넷 연결 필수
```

## 🚀 빠른 시작 (5분)

### Step 1: 프로젝트 복제

```bash
# 저장소 복제
git clone <repository-url>
cd AIMBP

# 또는 기존 프로젝트 디렉토리로 이동
cd f:/USB\ Drive/_웹어플/AIMBP
```

### Step 2: 환경 설정

```bash
# .env 파일 생성 (복사)
cp .env.example .env

# 에디터로 .env 파일 수정
# Windows: notepad .env
# Mac/Linux: nano .env
```

#### .env 파일 설정

```env
# 데이터베이스
DATABASE_URL=postgresql://aimbp:aimbp1234@db:5432/aimbp

# API 키
OPENAI_API_KEY=your-openai-api-key-here
SUNO_API_KEY=your-suno-api-key-here

# 백엔드 설정
BACKEND_CALLBACK_URL=http://localhost:8001
FILE_BASE_URL=http://localhost:8001

# 프론트엔드 설정
NEXT_PUBLIC_API_URL=http://localhost:8001/api/v1
```

### Step 3: 서버 시작

```bash
# Docker Compose로 모든 서비스 시작
docker-compose up -d

# 서비스 상태 확인
docker-compose ps

# 로그 확인 (선택사항)
docker-compose logs -f
```

### Step 4: 접속 확인

```
🌐 프론트엔드: http://localhost:3001
🔌 백엔드 API: http://localhost:8001
📊 API 문서: http://localhost:8001/docs
🗄️ 데이터베이스: localhost:5432
```

## 🐳 Docker Compose 서비스

### 포함된 서비스

```yaml
services:
  db:
    # PostgreSQL 16
    # 포트: 5432
    # 데이터: postgres_data 볼륨

  backend:
    # FastAPI 서버
    # 포트: 8001
    # 의존성: db

  frontend:
    # Next.js 서버
    # 포트: 3001
    # 의존성: backend
```

### 볼륨 (데이터 지속성)

```
postgres_data/          # 데이터베이스 데이터
uploads_data/           # 업로드된 음악/커버
frontend_node_modules/  # Node 의존성 캐시
```

## 🎮 기본 명령어

### 서비스 시작/중지

```bash
# 모든 서비스 시작 (백그라운드)
docker-compose up -d

# 특정 서비스만 시작
docker-compose up -d backend
docker-compose up -d frontend

# 모든 서비스 중지
docker-compose down

# 서비스 상태 확인
docker-compose ps

# 로그 확인
docker-compose logs              # 모든 서비스
docker-compose logs backend      # 백엔드만
docker-compose logs frontend     # 프론트엔드만
docker-compose logs -f           # 실시간 로그
```

### 서비스 재시작

```bash
# 모든 서비스 재시작
docker-compose restart

# 특정 서비스 재시작
docker-compose restart backend
docker-compose restart frontend

# 강제 재구축 및 시작
docker-compose up -d --build
```

### 데이터 관리

```bash
# 모든 데이터 삭제 (주의!)
docker-compose down -v

# 특정 볼륨만 삭제
docker volume rm aimbp_postgres_data

# 데이터베이스 마이그레이션
docker-compose exec backend alembic upgrade head

# 데이터베이스 초기화
docker-compose exec backend python -c "from app.core.database import init_db; init_db()"
```

## ❌ 문제 해결

### 포트가 이미 사용 중입니다

```bash
# 포트 사용 프로세스 확인
lsof -i :3001              # Mac/Linux
netstat -ano | findstr :3001   # Windows

# 포트 변경하기
# docker-compose.yml 수정
ports:
  - "3002:3000"  # 3001 → 3002로 변경
```

### 데이터베이스 연결 오류

```bash
# 데이터베이스 상태 확인
docker-compose ps db

# 데이터베이스 초기화
docker-compose down -v
docker-compose up -d

# 로그 확인
docker-compose logs db
```

### 프론트엔드 화이트 스크린

```bash
# 캐시 삭제
rm -rf frontend/.next
docker-compose restart frontend

# 의존성 재설치
docker-compose down
docker volume rm aimbp_frontend_node_modules
docker-compose up -d
```

### 백엔드 500 에러

```bash
# 백엔드 로그 확인
docker-compose logs backend

# 데이터베이스 마이그레이션 실행
docker-compose exec backend alembic upgrade head

# 백엔드 재시작
docker-compose restart backend
```

### API 연결 오류 (CORS)

```bash
# 브라우저 콘솔에서 CORS 에러 확인
# .env 파일에서 CORS_ORIGINS 설정 확인

# 또는 백엔드 로그 확인
docker-compose logs backend | grep -i cors
```

## 📊 헬스 체크

### 서비스 상태 확인

```bash
#!/bin/bash
# health-check.sh

echo "🔍 헬스 체크 시작..."

# 프론트엔드
echo -n "프론트엔드: "
curl -s http://localhost:3001 > /dev/null && echo "✅ OK" || echo "❌ 실패"

# 백엔드
echo -n "백엔드: "
curl -s http://localhost:8001/docs > /dev/null && echo "✅ OK" || echo "❌ 실패"

# 데이터베이스
echo -n "데이터베이스: "
docker-compose exec -T db pg_isready -U aimbp > /dev/null 2>&1 && echo "✅ OK" || echo "❌ 실패"

echo "✅ 헬스 체크 완료"
```

### 실행

```bash
chmod +x health-check.sh
./health-check.sh
```

## 📈 성능 최적화

### 메모리 설정

```yaml
# docker-compose.yml
services:
  backend:
    deploy:
      resources:
        limits:
          memory: 2G
        reservations:
          memory: 1G
```

### 데이터베이스 최적화

```bash
# PostgreSQL 최적화 설정
docker-compose exec db psql -U aimbp -d aimbp -c "
  ALTER SYSTEM SET shared_buffers='256MB';
  ALTER SYSTEM SET effective_cache_size='1GB';
  ALTER SYSTEM SET work_mem='16MB';
"

# 설정 적용 (재시작)
docker-compose restart db
```

### 정기 백업

```bash
# 데이터베이스 백업
docker-compose exec db pg_dump -U aimbp aimbp > backup-$(date +%Y%m%d).sql

# 볼륨 백업
docker run --rm \
  -v aimbp_postgres_data:/data \
  -v $(pwd):/backup \
  busybox tar czf /backup/postgres_data-$(date +%Y%m%d).tar.gz -C /data .
```

### 정기 청소

```bash
# 미사용 이미지 삭제
docker image prune -a

# 미사용 볼륨 삭제
docker volume prune

# 미사용 네트워크 삭제
docker network prune
```
