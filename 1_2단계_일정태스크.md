# AI Music Business Platform — Phase 1·2 일정 태스크

**작성일:** 2026-06-17
**총 기간:** Phase 1 (6주) + Phase 2 (6주) = 12주

---

## Phase 1 — 핵심 기반 (MVP)

> 목표: 음악 생성 → 저장 → 다운로드 기본 플로우 완성

---

### W1 — 프로젝트 세팅 (1주차)

#### 환경 구성
- [ ] Next.js 프로젝트 초기화 (App Router, TypeScript)
- [ ] FastAPI 프로젝트 초기화 (Python 3.11+)
- [ ] PostgreSQL 로컬 DB 세팅
- [ ] Docker Compose 구성 (frontend / backend / db)
- [ ] GitHub 레포 생성 및 브랜치 전략 수립
- [ ] `.env` 구조 설계 (API 키, DB URL 등)

#### 리뷰 체크
- [ ] Docker Compose `up` 후 전체 서비스 정상 기동 확인
- [ ] 환경 변수 누락 없이 로드되는지 확인
- [ ] 코드 리뷰: 폴더 구조, 네이밍 컨벤션

---

### W2 — 인증 시스템 (2주차)

#### Backend
- [ ] User 모델 설계 (id, email, password_hash, plan, created_at)
- [ ] 회원가입 API `POST /auth/register`
- [ ] 로그인 API `POST /auth/login` → JWT 발급
- [ ] JWT 토큰 검증 미들웨어
- [ ] 비밀번호 bcrypt 해시 처리
- [ ] Refresh Token 발급 및 갱신

#### Frontend
- [ ] 회원가입 페이지 UI
- [ ] 로그인 페이지 UI
- [ ] JWT 저장 (httpOnly 쿠키 또는 메모리)
- [ ] 인증 상태 전역 관리 (Context 또는 Zustand)
- [ ] 보호된 라우트 구현 (미로그인 시 리디렉션)

#### 리뷰 체크
- [ ] 비밀번호 평문 저장 여부 확인
- [ ] JWT 만료 처리 정상 동작 확인
- [ ] SQL Injection 방지 (파라미터 바인딩) 확인
- [ ] 로그인 실패 시 에러 메시지 노출 여부 확인
- [ ] API 응답 포맷 일관성 확인 (`{ success, data, error }`)

---

### W3 — AI 작사 기능 (3주차)

#### Backend
- [ ] OpenAI API 클라이언트 모듈 구현
- [ ] Claude API 클라이언트 모듈 구현
- [ ] 작사 생성 API `POST /lyrics/generate`
  - [ ] 입력: 주제, 분위기, 장르, 아티스트 스타일, 언어, AI 모델 선택
  - [ ] 출력: Verse / Chorus / Bridge / Hook 구조 JSON
- [ ] 생성 이력 저장 (Lyrics 모델: id, user_id, prompt, result, model, created_at)
- [ ] 생성 이력 조회 API `GET /lyrics`

#### Frontend
- [ ] 작사 생성 폼 UI (입력 필드 + AI 모델 선택)
- [ ] 생성 결과 표시 컴포넌트 (섹션별 구분)
- [ ] 생성 이력 목록 UI
- [ ] 로딩 상태 처리 (스트리밍 또는 폴링)

#### 리뷰 체크
- [ ] 프롬프트 인젝션 방어 로직 확인
- [ ] API 키 하드코딩 여부 확인
- [ ] 생성 실패 시 사용자 피드백 처리 확인
- [ ] 응답 토큰 비용 과다 방지 (max_tokens 제한) 확인
- [ ] 결과 JSON 파싱 오류 예외 처리 확인

### W4 — AI 작곡 기능 (4주차)

#### Backend
- [ ] Suno API 클라이언트 모듈 구현
- [ ] Udio API 클라이언트 모듈 구현
- [ ] 작곡 생성 API `POST /music/generate`
  - [ ] 입력: 가사, BPM, 장르, 분위기
  - [ ] 출력: 음원 파일 URL (MP3/WAV)
- [ ] 생성 작업 큐 처리 (Celery 또는 BackgroundTasks)
- [ ] Track 모델 설계 (id, user_id, lyrics_id, file_url, status, created_at)
- [ ] 생성 상태 조회 API `GET /music/{task_id}/status`

#### Frontend
- [ ] 작곡 생성 폼 UI
- [ ] 생성 진행 상태 폴링 UI
- [ ] 오디오 플레이어 컴포넌트
- [ ] 음원 다운로드 버튼

#### 리뷰 체크
- [ ] 비동기 작업 실패 시 상태 처리 확인
- [ ] 파일 URL 외부 노출 여부 확인 (인증된 사용자만 접근)
- [ ] 동시 생성 요청 제한 (Rate Limiting) 확인
- [ ] 생성 완료 알림 UX 확인

### W5 — S3 파일 관리 (5주차)

#### Backend
- [ ] AWS S3 버킷 설정 (버킷 정책, CORS)
- [ ] S3 업로드 모듈 구현 (boto3)
- [ ] Presigned URL 발급 API `GET /files/{file_id}/url`
- [ ] 파일 삭제 API `DELETE /files/{file_id}`
- [ ] 업로드 파일 크기 제한 (50MB)
- [ ] 지원 포맷 검증 (MP3, WAV, PNG, JPG)

#### Frontend
- [ ] 파일 업로드 컴포넌트 (드래그 앤 드롭)
- [ ] 업로드 진행 표시
- [ ] Presigned URL을 통한 다운로드

#### 리뷰 체크
- [ ] S3 버킷 퍼블릭 접근 차단 확인
- [ ] Presigned URL 만료 시간 설정 확인 (15분 이내)
- [ ] 파일명 경로 순회(Path Traversal) 방어 확인
- [ ] 파일 타입 서버 측 검증 확인 (MIME 타입)

## Phase 2 — 음악 제작 풀파이프라인

> 목표: 편곡·보컬·마스터링까지 AI 자동화

---

### W7 — AI 편곡 (7주차)

#### Backend
- [ ] 편곡 AI 서비스 연동 (MusicGen 또는 Stable Audio)
- [ ] 편곡 생성 API `POST /arrangement/generate`
  - [ ] 입력: 기준 트랙, 장르, 악기 선택 (드럼/베이스/기타/피아노/스트링/신스)
  - [ ] 출력: WAV, MP3, STEM 파일
- [ ] STEM 분리 처리 모듈
- [ ] Arrangement 모델 설계 (id, track_id, instruments, stems_url, created_at)

#### Frontend
- [ ] 편곡 설정 UI (악기 멀티셀렉트)
- [ ] STEM 트랙별 개별 플레이어
- [ ] 편곡 결과 다운로드 (전체 / STEM별)

#### 리뷰 체크
- [ ] STEM 파일 생성 성공률 확인 (90% 이상)
- [ ] 대용량 파일 처리 시 타임아웃 방어 확인
- [ ] 악기별 파일 URL 접근 권한 확인
- [ ] 편곡 생성 비용 추적 로그 확인

---

### W8 — AI 보컬 (8주차)

#### Backend
- [ ] 보컬 AI 서비스 연동 (TTS / 보컬 합성 API)
- [ ] 보컬 생성 API `POST /vocal/generate`
  - [ ] 입력: 가사, 성별 (남성/여성), 음색 (K-POP/ROCK/EDM/RAP), 언어
  - [ ] 출력: 보컬 WAV 파일
- [ ] 보컬 라이브러리 모델 (id, name, gender, genre, sample_url)
- [ ] 보컬 라이브러리 조회 API `GET /vocal/library`

#### Frontend
- [ ] 보컬 라이브러리 선택 UI (프리뷰 재생 포함)
- [ ] 보컬 생성 폼 UI
- [ ] 생성된 보컬 + 반주 믹스 미리듣기

#### 리뷰 체크
- [ ] 보컬 라이브러리 샘플 저작권 확인
- [ ] 다국어 입력 처리 (한국어/영어 최소) 확인
- [ ] 보컬 합성 음질 기준 충족 여부 확인
- [ ] 생성 실패 시 재시도 로직 확인

---

### W9 — AI 마스터링 (9주차)

#### Backend
- [ ] 마스터링 AI 서비스 연동
- [ ] 마스터링 API `POST /mastering/process`
  - [ ] 입력: 음원 파일, 타깃 플랫폼 (Spotify / Apple Music / YouTube Music)
  - [ ] 처리: Loudness 조정, EQ 최적화, Stereo Imaging, Limiter
  - [ ] 출력: 마스터링된 MP3/WAV
- [ ] 플랫폼별 Loudness 기준 상수 관리
  - [ ] Spotify: -14 LUFS
  - [ ] Apple Music: -16 LUFS
  - [ ] YouTube: -14 LUFS
- [ ] Mastering 모델 (id, track_id, platform, input_url, output_url, lufs, created_at)

#### Frontend
- [ ] 마스터링 설정 UI (플랫폼 선택)
- [ ] Before / After 파형 비교 UI
- [ ] 마스터링 결과 다운로드

#### 리뷰 체크
- [ ] LUFS 측정값 플랫폼 기준 내 확인
- [ ] 원본 파일 보존 여부 확인 (덮어쓰기 금지)
- [ ] 마스터링 처리 시간 기록 확인
- [ ] 결과물 음질 劣化 없음 확인

---

### W10 — 앨범 커버 생성 (10주차)

#### Backend
- [ ] GPT Image API 클라이언트 구현
- [ ] Stable Diffusion / Flux API 클라이언트 구현
- [ ] 커버 생성 API `POST /cover/generate`
  - [ ] 입력: 장르, 분위기, 키워드, AI 모델 선택
  - [ ] 출력: Cover Art, Banner, SNS 홍보 이미지 (1:1, 16:9, 9:16)
- [ ] Cover 모델 (id, user_id, prompt, image_url, size, created_at)

#### Frontend
- [ ] 커버 생성 폼 UI
- [ ] 이미지 결과 갤러리 (사이즈별)
- [ ] 이미지 다운로드 (PNG)
- [ ] 커버 이미지를 앨범에 연결하는 UI

#### 리뷰 체크
- [ ] 생성 이미지 저작권 정책 확인 (상업적 이용 가능 여부)
- [ ] 이미지 파일 크기 최적화 확인 (최대 5MB)
- [ ] 부적절한 이미지 필터링 (콘텐츠 정책) 확인
- [ ] 이미지 생성 비용 추적 확인
