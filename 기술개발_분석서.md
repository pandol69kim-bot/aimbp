## 🏗️ 2. 기술 스택 분석

### 2.1 Frontend 아키텍처

```
Next.js 15 (App Router)
├── TypeScript (타입 안정성)
├── Tailwind CSS + 디자인 시스템
│   ├── 색상: Primary(보라), Secondary(파랑), Accent(시안)
│   ├── 반응형: sm(640px), md(768px), lg(1024px)
│   └── 컴포넌트: Button, Card, Input, Dialog, Badge 등
├── React Query (TanStack Query)
│   ├── 서버 상태 관리
│   ├── 캐싱 전략
│   └── 자동 갱신 로직
└── API 통신
    ├── fetch 기반 래퍼 (api.ts)
    ├── Bearer Token 인증
    └── 에러 처리
```

**주요 특징**:
- ✅ 모던 App Router 사용 (Pages Router 대신)
- ✅ 강력한 타입 안정성 (TypeScript)
- ✅ 효율적인 상태 관리 (React Query)
- ✅ 반응형 설계 (모바일 우선)

**개선 기회**:
- 📌 TypeScript strict mode 전환 (현재 any 타입 남아있음)
- 📌 Error boundary 추가
- 📌 로깅 시스템 개선

### 2.2 Backend 아키텍처

```
FastAPI (Python)
├── 인증 (JWT, Bearer Token)
├── API 라우터
│   ├── /auth - 사용자 관리
│   ├── /music - 음악 생성/업로드
│   ├── /albums - 앨범 관리
│   ├── /lyrics - 가사 생성
│   ├── /cover - 커버 생성
│   └── /vocal - 보컬 생성
├── 서비스 레이어
│   ├── ai_music.py (Suno, Mureka, Udio)
│   ├── lyrics_service.py (GPT 기반 가사)
│   ├── vocal_service.py (TTS)
│   └── album_service.py (앨범 관리)
├── 데이터 레이어 (SQLAlchemy 2.0)
│   ├── User (사용자)
│   ├── Track (음악 트랙)
│   ├── Album (앨범)
│   ├── Lyrics (가사)
│   ├── Cover (커버)
│   └── 기타
└── 인프라
    ├── PostgreSQL 16
    ├── 파일 저장소 (/app/uploads)
    └── 캐싱 (선택사항)
```

**주요 특징**:
- ✅ 마이크로서비스 스타일의 서비스 분리
- ✅ 비동기 처리 (async/await)
- ✅ 명확한 API 응답 구조
- ✅ 상세한 에러 처리

**개선 기회**:
- 📌 Database connection pooling
- 📌  캐싱 전략 (Redis)
- 📌 로깅 및 모니터링
- 📌 배치 작업 처리 (Celery)

## 🎯 4. 주요 기술 결정

### 4.1 AI 서비스 선택

#### Suno AI ✅ (선택됨)
- **장점**: 가장 높은 음질, 빠른 생성
- **단점**: API 콜백 복잡성
- **상태**: 완전히 통합됨 (Polling 방식)

#### Mureka AI ⚠️ (Mock 모드)
- **장점**: 독특한 음악 스타일
- **문제**: API 키 검증 이슈
- **현황**: Mock 데이터로 임시 작동

#### Udio AI ❌ (미구현)
- **상태**: 향후 구현 예정

**결론**: Suno AI에 집중 (높은 사용성)

### 4.2 데이터베이스 설계

**선택**: PostgreSQL 16 + SQLAlchemy 2.0

```
주요 테이블:
├── users (사용자)
├── tracks (음악 트랙)
├── albums (앨범)
├── lyrics (가사)
├── covers (앨범 커버)
├── download_tokens (다운로드 관리)
└── download_history (이력 추적)
```

**마이그레이션 현황**:
- ✅ Alembic 설정 완료
- ✅ artist_name 컬럼 추가 (MP3 업로드용)
- ✅ 안전한 마이그레이션 전략

### 4.3 인증 방식

**선택**: JWT (Bearer Token)

```
Flow:
1. 회원가입 → 계정 생성
2. 로그인 → access_token + refresh_token
3. API 호출 → Authorization: Bearer <token>
4. 토큰 갱신 → refresh_token으로 신규 발급
```

**보안**:
- ✅ 비밀번호 해싱 (bcrypt)
- ✅ 토큰 만료 시간 설정
- ✅ CORS 설정
