# MP3 Playlist for YouTube - Claude Instructions

## UI/UX 표준 디자인 시스템 (MANDATORY)

모든 프론트엔드 개발은 정의된 디자인 시스템을 따라야 합니다.

### 참고 문서

- **전체 가이드**: `DESIGN_SYSTEM.md` - 색상, 타이포그래피, 컴포넌트, 레이아웃, 접근성 등 상세 가이드
- **빠른 참고**: `UI-QUICK-REFERENCE.md` - 자주 사용하는 패턴과 체크리스트

### 디자인 시스템 핵심 요소

#### 색상 팔레트
- **Primary** (보라): #7c3aed - 주요 행동
- **Secondary** (파란): #3b82f6 - 보조 행동
- **Accent** (시안): #06b6d4 - 강조
- **Background**: #0f0f0f (매우 어두움)

#### 컴포넌트 재사용
- `frontend/src/components/ui/` - 기본 컴포넌트 (Button, Card, Input 등)
- 새 컴포넌트 생성 전에 기존 것 확인 필수

#### 체크리스트
- 모든 상호작용 요소는 포커스 상태 필수
- 색상은 정의된 팔레트에서만 선택
- 반응형 디자인 필수 (모바일, 태블릿, 데스크톱)
- 접근성 요구사항 충족 (WCAG AA)

## 작업 프로세스 (MANDATORY)

모든 작업은 다음 3단계를 따릅니다:

### Step 1: 작업 지시 받기
작업 지시를 받으면 **작업 계획**을 먼저 저장합니다.

### Step 2: 작업 계획 저장
작업 계획 파일을 생성합니다.

**파일명 형식:**
```
_docs/작업계획_{날짜}_{시간}_{작업내용요약}.md
```

- `{날짜}`: `YYYYMMDD` 형식 (예: `20260615`)
- `{시간}`: `HHMM` 형식 (예: `1430`)
- `{작업내용요약}`: 영문 또는 한글 키워드, 공백 대신 `-` 사용

**예시:** `_docs/작업계획_20260619_1200_MP3업로드기능개발.md`

**계획 문서 내용 구조:**
```markdown
# 작업 계획

- **작업명**: [작업 제목]
- **시작시간**: YYYY-MM-DD HH:MM
- **예상 소요시간**: X분/시간
- **작업자**: Claude

## 작업 목표

(이 작업으로 달성할 목표)

## 구현 계획

1. [첫 번째 단계]
2. [두 번째 단계]
3. [세 번째 단계]

## 변경될 파일

- `파일경로1`: 변경 내용
- `파일경로2`: 변경 내용

## 예상 결과

(완료 후 어떤 결과물이 나올 것인지)
```

### Step 3: 작업 수행
계획에 따라 실제 작업을 진행합니다.

### Step 4: 작업 내역 저장 (MANDATORY)
모든 작업이 완료된 후 반드시 `_docs` 폴더에 **작업 내역** 문서를 작성합니다.

**파일명 형식:**
```
_docs/작업내역_{날짜}_{시간}_{작업내용요약}.md
```

- `{날짜}`: `YYYYMMDD` 형식 (예: `20260615`)
- `{시간}`: `HHMM` 형식 (예: `1430`)
- `{작업내용요약}`: 영문 또는 한글 키워드, 공백 대신 `-` 사용

**예시:** `_docs/작업내역_20260619_1230_MP3업로드기능개발.md`

**작업 내역 문서 내용 구조:**
```markdown
# 작업 내역

- **날짜**: YYYY-MM-DD HH:MM
- **작업자**: Claude
- **소요시간**: X분

## 작업 내용

(수행한 작업을 간결하게 서술)

## 변경된 파일

- `파일경로`: 변경 내용 요약

## 완료 항목

- ✅ [완료한 항목 1]
- ✅ [완료한 항목 2]
- ✅ [완료한 항목 3]

## 주요 결정 사항

(중요한 설계 결정이나 선택 이유가 있으면 기록)

## 테스트 결과

(작업 결과 테스트 상황)

## 작업 커맨드 로그

- `작업시간` : 커맨드
```
