## 1. 프로젝트 개요

### 1.1 비전

**"1인 AI 뮤직 레이블"**

누구나 AI를 이용하여 작사 → 작곡 → 편곡 → 음원 제작 → 발매 → 유통 → 정산 → 수익화까지 수행하고, 모든 저작권과 스트리밍 수익을 직접 소유하는 글로벌 음악 비즈니스 플랫폼.

### 1.2 핵심 가치 제안

- **비용 효율성**: 전통 음악 제작 비용 90% 절감
- **시간 단축**: 음원 출시 시간 단축 (48시간 이내)
- **지식 진입장벽 제거**: AI 기술으로 음악 제작 전 과정 자동화
- **완전한 지분 소유**: 생성 음악의 100% 저작권 보유
- **글로벌 유통**: Spotify, Apple Music, YouTube Music 등 글로벌 플랫폼 자동 유통

### 1.3 타겟 사용자

- 개인 크리에이터 (Singer/Songwriter 꿈나무)
- 독립 뮤지션 (Independent Artists)
- 음악 프로듀서 (Music Producer)
- 콘텐츠 크리에이터 (유튜브, 틱톡 크리에이터)
- 뮤직 레이블 (작은 레이블, 스타트업)

### 1.4 핵심 KPI

| 지표 | 목표 |
|------|------|
| 월 음원 생성 수 | 100만 곡 |
| 월 발매 수 | 10만 곡 |
| 활성 사용자 | 50만 명 |
| 유통 음원 누적 | 1,000만 곡 |
| 월 정산 규모 | 100억 원 |
| MRR (월간 반복 수익) | 10억 원 이상 |

### 3.2 데이터 흐름

**음악 생성 흐름:**
```
User Input → Lyrics Service (AI) → Music Service (Suno/Udio) 
  → Arrangement Service → Mastering Service → S3 Upload 
  → Database Record → Download Available
```

**앨범 발매 흐름:**
```
User Album Setup → Cover Generation (AI Image) → Metadata Input 
  → Distribution API → Multiple Platforms (Spotify, Apple, etc) 
  → Revenue Tracking → Dashboard Display
```

### 3.3 마이크로서비스 고려사항 (향후)

현재 모놀리식 구조이나, 향후 확장성을 위해:
- **AI 작사 서비스**: 별도 마이크로서비스로 분리 가능
- **음악 생성 서비스**: Suno/Udio API 호출 전담
- **파일 처리 서비스**: S3 업로드/다운로드 및 변환 담당
- **유통 서비스**: 다양한 스트리밍 플랫폼과의 연동 담당

```python
from sqlalchemy import Column, String, DateTime, ForeignKey
from sqlalchemy.orm import relationship
from datetime import datetime

class Track(Base):
    __tablename__ = "tracks"

    id = Column(String, primary_key=True, index=True)
    user_id = Column(String, ForeignKey("users.id"), nullable=False)
    title = Column(String(200), nullable=False)
    genre = Column(String(100), nullable=False)
    mood = Column(String(100), nullable=False)
    music_file_url = Column(String, nullable=True)
    cover_url = Column(String, nullable=True)
    created_at = Column(DateTime, default=datetime.utcnow)
    updated_at = Column(DateTime, default=datetime.utcnow, onupdate=datetime.utcnow)

    # Relationships
    user = relationship("User", back_populates="tracks")
    lyrics = relationship("Lyrics", back_populates="track", uselist=False)
```

```python
    track_create: TrackCreate,
    current_user: User = Depends(get_current_user),
    db: AsyncSession = Depends(get_db),
) -> TrackResponse:
    # 비즈니스 로직
    db_track = Track(**track_create.dict(), user_id=current_user.id)
    db.add(db_track)
    await db.commit()
    await db.refresh(db_track)
    return db_track

@router.get("/{track_id}", response_model=TrackResponse)
async def get_track(
    track_id: str,
    db: AsyncSession = Depends(get_db),
) -> TrackResponse:
    # 조회 로직
    track = await db.get(Track, track_id)
    if not track:
        raise HTTPException(status_code=404, detail="Track not found")
    return track
```

#### 6.2.5 서비스 레이어 패턴
```python
# app/services/ai_lyrics.py
class LyricsService:
    def __init__(self, openai_client, claude_client):
        self.openai = openai_client
        self.claude = claude_client

    async def generate_lyrics(
        self,
        topic: str,
        mood: str,
        language: str,
    ) -> dict:
        # OpenAI를 이용한 가사 생성
        prompt = self._build_prompt(topic, mood, language)
        response = await self.openai.ChatCompletion.acreate(
            model="gpt-4",
            messages=[{"role": "user", "content": prompt}],
        )
        return self._parse_response(response)

    async def refine_lyrics(
        self,
        lyrics: str,
    ) -> str:
        # Claude를 이용한 가사 개선
        refined = await self.claude.messages.create(
            model="claude-3-opus",
            messages=[{"role": "user", "content": f"Refine these lyrics: {lyrics}"}],
        )
        return refined.content[0].text
```
